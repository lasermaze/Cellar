  <name>Task 1: Implement GitHubAuthService with RS256 JWT, token exchange, cache, and credential loading</name>
  <files>Sources/cellar/Core/GitHubAuthService.swift</files>
  <action>
Create `Sources/cellar/Core/GitHubAuthService.swift` with `import Foundation` and `import Security`. Make it a `final class` (not struct) because it holds mutable cached token state. Use an `NSLock` to serialize cache access for thread safety (per research recommendation — Vapor routes may call from concurrent threads).

**Public API:**

```swift
final class GitHubAuthService {
    static let shared = GitHubAuthService()  // singleton for convenience
    func getToken() -> GitHubAuthResult      // main entry point for phases 15/16
}
```

**Internal sections (implement in this order):**

1. **Cached token state** (private):
   - `private var cachedToken: String?`
   - `private var tokenExpiry: Date?`
   - `private let lock = NSLock()`

2. **`getToken() -> GitHubAuthResult`** (public):
   - Acquire lock, check if `cachedToken` exists AND `tokenExpiry > Date().addingTimeInterval(5 * 60)` (5-minute buffer = effective 55-min TTL per AUTH-02)
   - If cache hit, return `.token(cachedToken!)`
   - If cache miss, call `refreshToken()`
   - On success, store token + expiry in cache, return `.token(...)`
   - On `GitHubAuthError.credentialsNotConfigured`, return `.unavailable(reason: "GitHub App credentials not configured — set GITHUB_APP_ID, GITHUB_INSTALLATION_ID, and GITHUB_APP_KEY_PATH in ~/.cellar/.env or environment")`
   - On any other error, return `.unavailable(reason: error.localizedDescription)`
   - Release lock in all paths (use defer)

3. **`refreshToken() throws -> (token: String, expiresAt: Date)`** (private):
   - Load credentials via `loadCredentials()`
   - Generate JWT via `makeJWT(appID:pemString:)`
   - Exchange JWT for installation token via `fetchInstallationToken(jwt:installationID:)`
   - Return token + expiry tuple

4. **`makeJWT(appID: String, pemString: String) throws -> String`** (private):
   - **PEM to DER:** Strip `-----BEGIN RSA PRIVATE KEY-----` and `-----END RSA PRIVATE KEY-----` headers (also defensively strip PKCS#8 `-----BEGIN PRIVATE KEY-----` / `-----END PRIVATE KEY-----`). Remove all `\r`, `\n`, whitespace. Base64-decode to Data. Throw `.invalidPEM` if decode fails.
   - **Create SecKey:** Build attributes dict `[kSecAttrKeyType: kSecAttrKeyTypeRSA, kSecAttrKeyClass: kSecAttrKeyClassPrivate]`. Call `SecKeyCreateWithData(der as CFData, attrs as CFDictionary, &cfError)`. Throw `.keyCreationFailed(cfError description)` if nil.
   - **Build JWT header + payload:**
     - Header: `{"alg":"RS256","typ":"JWT"}` — JSON-encode, base64url-encode
     - Payload: `{"iss": appID, "iat": now - 60, "exp": now + 510}` — use `[String: Any]` dictionary, serialize with `JSONSerialization.data(withJSONObject:)`, base64url-encode. `iat = now - 60` for clock skew protection. `exp = now + 510` (8.5 min, under GitHub's 10-min max).
   - **Sign:** `let message = "\(header).\(payload)"`, encode to UTF-8 Data. Call `SecKeyCreateSignature(secKey, .rsaSignatureMessagePKCS1v15SHA256, messageData as CFData, &cfError)`. Throw `.signFailed(cfError description)` if nil.
   - **Assemble:** Return `"\(header).\(payload).\(base64url(signature as Data))"`

5. **`base64url(_ data: Data) -> String`** (private helper):
   ```swift
   data.base64EncodedString()
       .replacingOccurrences(of: "+", with: "-")
       .replacingOccurrences(of: "/", with: "_")
       .replacingOccurrences(of: "=", with: "")
   ```

6. **`fetchInstallationToken(jwt: String, installationID: String) throws -> (token: String, expiresAt: Date)`** (private):
   - Build URLRequest: `POST https://api.github.com/app/installations/{installationID}/access_tokens`
   - Headers: `Authorization: Bearer {jwt}`, `Accept: application/vnd.github+json`, `X-GitHub-Api-Version: 2022-11-28`
   - Use the same DispatchSemaphore + ResultBox pattern from AIService.callAPI (copy the pattern, don't import AIService — it's a static struct with private methods)
   - Parse HTTP status: if >= 400, throw `.httpError(statusCode:body:)` with response body as string for debugging
   - Decode `InstallationTokenResponse` from response data
   - Parse `expiresAt` with `ISO8601DateFormatter()`. Throw `.invalidExpiryDate` if parsing fails.
   - Return (token, expiresAt date)

7. **`loadCredentials() throws -> GitHubCredentials`** (private):
   - Load environment using the SAME pattern as `AIService.loadEnvironment()` — read `ProcessInfo.processInfo.environment`, merge with `~/.cellar/.env` file (process env takes precedence). Copy the .env parsing logic (10 lines) rather than trying to share it across types — keeping it self-contained is fine for two call sites.
   - **PEM key resolution** (priority cascade):
     a. `GITHUB_APP_KEY_PATH` env var → read file at that path
     b. Bundled resource — use RecipeEngine's multi-strategy pattern:
        - `Bundle.main.url(forResource: "github-app", withExtension: "pem")` first
        - Then `URL(fileURLWithPath: FileManager.default.currentDirectoryPath).appendingPathComponent("Sources/cellar/Resources/github-app.pem")` for development
     c. If neither exists, throw `.credentialsNotConfigured`
   - **App ID:** `env["GITHUB_APP_ID"]` ?? bundled `github-app.json`'s `appID`
   - **Installation ID:** `env["GITHUB_INSTALLATION_ID"]` ?? bundled `github-app.json`'s `installationID`
   - **Bundled config loading:** Use same multi-strategy pattern for `github-app.json` (Bundle.main first, then cwd-relative)
   - Guard that both `appID` and `installationID` are non-empty. Throw `.credentialsNotConfigured` if either is empty.
   - Return `GitHubCredentials(appID:installationID:pemString:)`

8. **`memoryRepo` computed property** (public):
   - `loadEnvironment()["CELLAR_MEMORY_REPO"] ?? CellarPaths.defaultMemoryRepo`
   - Returns the `owner/repo` string for collective memory API calls in later phases.

**Critical implementation notes from research:**
- Do NOT cache the JWT — generate fresh each time `refreshToken()` is called. JWTs are valid 10 min max, and RSA signing is fast (local operation).
- Use `JSONSerialization` (not JSONEncoder) for JWT payload because the payload mixes String and Int values (`"iss"` is String, `"iat"` and `"exp"` are Int). JSONEncoder requires homogeneous Codable types.
- The `base64url` helper is 3 lines — standard base64 with `+` → `-`, `/` → `_`, strip `=` padding.
- GitHub App keys use PKCS#1 format (`BEGIN RSA PRIVATE KEY`), not PKCS#8 (`BEGIN PRIVATE KEY`). Strip the correct headers.
- Set `iat = now - 60` (60 seconds in the past) to handle clock skew — this is GitHub's recommended value.
