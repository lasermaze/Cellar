  <name>Task 1: Create GitHubModels.swift with all Codable types and error enum</name>
  <files>Sources/cellar/Models/GitHubModels.swift</files>
  <action>
Create `Sources/cellar/Models/GitHubModels.swift` with these types:

1. **`GitHubAppConfig`** — Codable struct matching `github-app.json` structure:
   - `appID: String` (CodingKey: `app_id`) — accepts both numeric App ID and string Client ID per research
   - `installationID: String` (CodingKey: `installation_id`)

2. **`GitHubCredentials`** — non-Codable struct holding resolved credentials:
   - `appID: String`
   - `installationID: String`
   - `pemString: String`

3. **`InstallationTokenResponse`** — Codable struct for GitHub API response:
   - `token: String`
   - `expiresAt: String` (CodingKey: `expires_at`) — ISO 8601 date string

4. **`GitHubAuthResult`** — enum mirroring AIProvider pattern:
   - `case token(String)` — valid installation access token
   - `case unavailable(reason: String)` — credentials missing or misconfigured

5. **`GitHubAuthError`** — enum conforming to Error and LocalizedError:
   - `case credentialsNotConfigured` — app ID or installation ID empty/missing
   - `case invalidPEM` — PEM file cannot be parsed (base64 decode failure)
   - `case keyCreationFailed(String)` — SecKeyCreateWithData returned nil (store CFError description as String)
   - `case signFailed(String)` — SecKeyCreateSignature returned nil
   - `case encodingFailed` — UTF-8 encoding of JWT message failed
   - `case invalidExpiryDate` — expires_at field not ISO 8601
   - `case httpError(statusCode: Int, body: String)` — GitHub API returned error
   - Provide `errorDescription` for each case via LocalizedError

Follow existing model patterns: `import Foundation`, plain structs, explicit CodingKeys where JSON uses snake_case.
