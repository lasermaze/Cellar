  <action>
**CellarConfig.swift** -- Add two things:
1. Add `var contributeMemory: Bool?` field with CodingKey `"contribute_memory"`. Nil = not asked, true = opted in, false = declined.
2. Add `static func save(_ config: CellarConfig) throws` method: encode with JSONEncoder (prettyPrinted + sortedKeys), createDirectory at CellarPaths.base, write to CellarPaths.configFile with .atomic option.

**CollectiveMemoryWriteService.swift** -- Create new file as a stateless struct with all static methods, mirroring CollectiveMemoryService patterns:

1. `static func push(record: SuccessRecord, gameName: String, wineURL: URL)` -- public entry point. Never throws. All errors caught and logged internally.
   - Get auth token via `GitHubAuthService.shared.getToken()` -- return silently if .unavailable
   - Detect wine version and flavor using same approach as CollectiveMemoryService (run `wine --version` via Process, check GPTK paths). Copy the `detectWineVersion` and `detectWineFlavor` private helpers from CollectiveMemoryService (they are private there, so duplicate them here -- or extract to a shared helper if cleaner).
   - Build EnvironmentFingerprint.current(wineVersion:wineFlavor:)
   - Transform SuccessRecord to CollectiveMemoryEntry: map environment, dllOverrides, registry directly; launchArgs = [], setupDeps = []; reasoning = resolutionNarrative ?? ""; engine/graphicsApi from record; confirmations = 1; lastConfirmed = ISO8601DateFormatter().string(from: Date()); schemaVersion = 1; environmentHash = fingerprint.computeHash()
   - Call pushEntry() with the constructed entry

2. `private static func pushEntry(entry: CollectiveMemoryEntry, token: String)` -- the GET + merge + PUT flow:
   - Build slug from entry.gameName using slugify()
   - GET `https://api.github.com/repos/{memoryRepo}/contents/entries/{slug}.json` with Accept: `application/vnd.github+json` (NOT raw) and Authorization Bearer token. timeoutInterval = 5.
   - If 404: PUT new file (no sha field) with commit message "Add {gameName} entry"
   - If 200: decode GitHubContentsResponse (sha + base64 content), strip newlines from base64 before Data(base64Encoded:), decode [CollectiveMemoryEntry]
   - Merge: find existing entry with same environmentHash -> if found, create new CollectiveMemoryEntry with confirmations+1 and updated lastConfirmed (struct has let fields, must construct new value). If not found, append new entry.
   - Commit message: "Update {gameName} (+1 confirmation)" for increment, "Update {gameName} (new environment)" for append
   - PUT: encode entries array, base64EncodedString(), build body with message + content + sha (omit sha key entirely for new file), PUT to same URL with Content-Type application/json. timeoutInterval = 5.
   - If PUT returns 409: re-fetch (GET again), re-merge, PUT again with new sha. ONE retry only. If still 409, log and return.

3. `private static func performRequest(request: URLRequest) -> (data: Data, statusCode: Int)?` -- copy the DispatchSemaphore + ResultBox pattern from CollectiveMemoryService exactly.

4. `private static func logPushEvent(_ level: String, gameId: String, _ message: String)` -- append one line to CellarPaths.logsDir/memory-push.log. Format: `{ISO8601} {LEVEL} {gameId} {message}\n`. Use FileHandle for append, create file if it does not exist. Create logsDir if needed.

5. Internal types: `GitHubContentsResponse: Codable` with sha + content fields. `GitHubPutBody: Codable` or use JSONSerialization dict for the PUT body (simpler since sha is conditionally included).

Key pitfalls to avoid:
- Strip `\n` from base64 content before decoding (GitHub wraps at 60 chars per RFC 2045)
- CollectiveMemoryEntry has `let` fields -- construct new value for confirmation increment, do not try to mutate
- Total write operation must complete within ~10s (5s per request is fine with at most 2-3 requests)
- Never throw from push() -- wrap everything in do/catch, log errors via logPushEvent
