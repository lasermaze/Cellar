2. **Progressive system prompts** in `_generateVariants()` — Replace the single `systemPrompt` with a level-dependent prompt:

**Level 1** (current behavior): Env vars and WINEDLLOVERRIDES only. Use the existing prompt as-is.

**Level 2**: Add wiretricks verbs + DLL overrides. System prompt adds:
```
You may also suggest:
- "install_winetricks" actions with a verb name (e.g., "d3dx9", "vcrun2019", "dsound")
- "set_dll_override" actions with DLL name and mode

Return actions as an array of typed objects:
{"type": "set_env", "key": "...", "value": "..."}
{"type": "set_dll_override", "dll": "...", "mode": "..."}
{"type": "install_winetricks", "verb": "..."}
```

**Level 3**: Add everything. System prompt adds:
```
You may also suggest:
- "place_dll" actions to download and install DLL replacements (e.g., cnc-ddraw for DirectDraw games)
- "set_registry" actions for Wine registry modifications

Known DLL replacements available for auto-download:
- cnc-ddraw: DirectDraw replacement for classic 2D games (fixes "DirectDraw Init Failed" errors)

Return actions as typed objects:
{"type": "place_dll", "dll": "cnc-ddraw", "target": "game_dir"}
{"type": "set_registry", "key": "HKCU\\Software\\...", "value_name": "...", "data": "dword:00000001"}
```

All levels should include:
- Game context: game era, installer type, Wine translation stack
- The `"reasoning"` field requirement (AI reasons first, actions follow)
- Instruction to return JSON only

The key change is: **Level 1 prompt explicitly prohibits registry/DLL/winetricks. Level 2 allows winetricks. Level 3 allows everything including DLL placements.**

Build the prompt using a helper function `buildVariantSystemPrompt(level:)` for readability.

4. **Expand parseWineFix()** to handle new action types from AI:

Add cases for the new action types (the AI now returns typed `actions` arrays at Level 2+):
```swift
case "place_dll":
    let dllName = arg1  // maps to KnownDLLRegistry name
    guard !dllName.isEmpty else { return nil }
    let target: DLLPlacementTarget = (arg2 == "system32") ? .system32 : .gameDir
    return .placeDLL(dllName, target)

    guard !arg1.isEmpty else { return nil }
    return .setRegistry(arg1, arg2, /* need arg3 — see below */)
```

Since the current `parseWineFix` only takes `arg1` and `arg2`, and `setRegistry` needs 3 args (key, valueName, data), update the function signature to accept the full dict:

This new overload handles the action-object format: `dict["type"]` determines the action, other keys provide args. Keep the old `parseWineFix(fixType:arg1:arg2:)` for backward compat with the diagnose() response format.
