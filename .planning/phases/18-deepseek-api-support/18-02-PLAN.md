**AgentLoop.swift refactor:**

Replace the `apiKey` and `model` properties with a single `provider` property of type `AgentLoopProvider`. Update the init:

```swift
struct AgentLoop {
    var provider: AgentLoopProvider  // var because protocol methods are mutating
    let maxIterations: Int
    let maxTokens: Int
    let budgetCeiling: Double
    let onOutput: (@Sendable (AgentEvent) -> Void)?

    init(
        provider: AgentLoopProvider,

        onOutput: (@Sendable (AgentEvent) -> Void)? = nil
    )
}
```

Note: `tools` and `systemPrompt` are no longer AgentLoop properties — they're passed to the provider at construction time by AIService.

**Refactor `run()` method:**

1. Replace hardcoded pricing constants (lines 148-150) with:
   ```swift
   let pricing = provider.pricingPerToken()
   let inputPricePerToken = pricing.input
   let outputPricePerToken = pricing.output
   ```

2. Replace `messages` array management — AgentLoop no longer holds `[AnthropicToolRequest.Message]`. Instead:
   - Initial user message: `provider.appendUserMessage(initialMessage)`
   - API call: `let response = try provider.callWithRetry(maxTokens: currentMaxTokens, emit: emit)`
   - Token accumulation: use `response.inputTokens` and `response.outputTokens`
   - Text collection: iterate `response.textBlocks`
   - Stop reason handling: switch on `response.stopReason` (`.endTurn`, `.toolUse`, `.maxTokens`, `.other`)

3. For `end_turn` handling:
   - Check `response.textBlocks.isEmpty && response.toolCalls.isEmpty` for empty response
   - For continuation: `provider.appendAssistantResponse(response)` then `provider.appendUserMessage("continuation text")`

4. For `tool_use` handling:
   - `provider.appendAssistantResponse(response)` to store assistant turn
   - Execute tools: iterate `response.toolCalls`, call `toolExecutor(name, input)`
   - Collect results: `provider.appendToolResults(results)`
   - Budget warning injection: append as a user message via `provider.appendUserMessage(warnMsg)` (no longer a ToolContentBlock in the results array — simplification)

5. For `max_tokens` handling:
   - Check `!response.toolCalls.isEmpty` for incomplete tool use (instead of checking ToolContentBlock)
   - Continuation: `provider.appendAssistantResponse(response)` then `provider.appendUserMessage("Your response was truncated. Continue.")`

6. For budget halt: `provider.appendAssistantResponse(response)` then `provider.appendUserMessage(stopMsg)` then `try provider.callWithRetry(...)` for final save call.

7. **DELETE** private methods: `callAnthropic()`, `callAnthropicWithRetry()`, `callAPI()` — all moved to providers in Plan 01.

**AIService.swift changes:**

1. **`detectProvider()`** — rewrite to support AI_PROVIDER config:
   ```swift
   static func detectProvider() -> AIProvider {
       let env = loadEnvironment()
       let configProvider = CellarConfig.load().aiProvider ?? env["AI_PROVIDER"]

       switch configProvider?.lowercased() {
       case "deepseek":
           if let key = env["DEEPSEEK_API_KEY"], !key.isEmpty { return .deepseek(apiKey: key) }
           return .unavailable  // Deepseek requested but no key
       case "claude", "anthropic":
           if let key = env["ANTHROPIC_API_KEY"], !key.isEmpty { return .anthropic(apiKey: key) }
           return .unavailable  // Claude requested but no key
       default:
           // Auto-detect: check which keys are present
           let hasAnthropic = env["ANTHROPIC_API_KEY"].map { !$0.isEmpty } ?? false
           let hasDeepseek = env["DEEPSEEK_API_KEY"].map { !$0.isEmpty } ?? false
           if hasAnthropic { return .anthropic(apiKey: env["ANTHROPIC_API_KEY"]!) }
           if hasDeepseek { return .deepseek(apiKey: env["DEEPSEEK_API_KEY"]!) }
           // Legacy: check OpenAI key
           if let key = env["OPENAI_API_KEY"], !key.isEmpty { return .openai(apiKey: key) }
           return .unavailable
       }
   }
   ```

2. **`runAgentLoop()`** — update to create provider based on detected provider:
   - For `.anthropic(let apiKey)`: create `AnthropicAgentProvider(apiKey: apiKey, model: "claude-sonnet-4-6", tools: tools, systemPrompt: systemPrompt)`
   - For `.deepseek(let apiKey)`: create `DeepseekAgentProvider(apiKey: apiKey, tools: tools, systemPrompt: systemPrompt)`
   - For `.openai` or `.unavailable`: return `.unavailable`
   - Then create `AgentLoop(provider: provider, ...)` and call `loop.run(...)`
   - Note: the existing `guard case .anthropic` must be replaced with a switch that handles both `.anthropic` and `.deepseek`

3. **`makeAPICall()`** — add `.deepseek` case for simple (non-agent) API calls:
   - Route to a new `callDeepseek(apiKey:systemPrompt:userMessage:)` private static method
   - Uses `OpenAIRequest` (the existing simple OpenAI type, NOT the new tool-use types) with model `"deepseek-chat"` and URL `https://api.deepseek.com/v1/chat/completions`, Bearer auth
   - Decodes `OpenAIResponse` (existing type — same format as OpenAI simple responses)
   - This reuses the same pattern as `callOpenAI()` but with different URL and auth

4. **Provider-specific error messages** (DSPK-03):
   - When `detectProvider()` returns `.unavailable` and a specific provider was requested, the error should name the provider. Update `runAgentLoop()`:
     ```swift
     case .unavailable:
         let env = loadEnvironment()
         let requested = CellarConfig.load().aiProvider ?? env["AI_PROVIDER"]
         if requested?.lowercased() == "deepseek" {
             return .failed("Deepseek API key not configured. Set DEEPSEEK_API_KEY in ~/.cellar/.env or environment.")
         } else if requested != nil {
             return .failed("Anthropic API key not configured. Set ANTHROPIC_API_KEY in ~/.cellar/.env or environment.")
         }
         return .unavailable
     ```
   - Apply same pattern to `diagnose()` and `generateVariants()` methods.
