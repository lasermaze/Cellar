2. **`AgentLoopProvider`** protocol:
   ```swift
   protocol AgentLoopProvider {
       var modelName: String { get }
       func pricingPerToken() -> (input: Double, output: Double)
       mutating func appendUserMessage(_ text: String)
       mutating func appendAssistantResponse(_ response: AgentLoopProviderResponse)
       mutating func appendToolResults(_ results: [(id: String, content: String, isError: Bool)])
       mutating func callWithRetry(maxTokens: Int, emit: (AgentEvent) -> Void) throws -> AgentLoopProviderResponse
   }
   ```
   The provider owns the message array entirely. AgentLoop never holds provider-specific message types.

3. **`AnthropicAgentProvider`** struct implementing the protocol:
   - Init: `apiKey: String, model: String, tools: [ToolDefinition], systemPrompt: String`
   - Stores `var messages: [AnthropicToolRequest.Message]` internally
   - `appendUserMessage`: appends `Message(role: "user", content: .text(text))`
   - `appendAssistantResponse`: converts `AgentLoopProviderResponse` back to Anthropic format — text blocks become `.text()` content blocks, tool calls become `.toolUse()` content blocks. Appends as `Message(role: "assistant", content: .blocks(...))`
   - `appendToolResults`: converts to `ToolContentBlock.toolResult` blocks, appends as `Message(role: "user", content: .blocks(resultBlocks))`
   - `callWithRetry`: Extract the existing retry logic from AgentLoop.callAnthropicWithRetry (3 attempts, backoff [1s, 2s, 4s], retry on 5xx/429/network, fatal on other 4xx). Builds `AnthropicToolRequest`, calls Anthropic API at `https://api.anthropic.com/v1/messages`, decodes `AnthropicToolResponse`, translates to `AgentLoopProviderResponse`:
     - `stop_reason: "end_turn"` → `.endTurn`
     - `stop_reason: "tool_use"` → `.toolUse`
     - `stop_reason: "max_tokens"` → `.maxTokens`
     - Other → `.other(value)`
   - Uses same `callAPI()` pattern with ResultBox + DispatchSemaphore + URLSession.shared
   - `pricingPerToken`: returns from static pricing map

4. **`DeepseekAgentProvider`** struct implementing the protocol:
   - Init: `apiKey: String, model: String = "deepseek-chat", tools: [ToolDefinition], systemPrompt: String`
   - Stores `var messages: [OpenAIToolRequest.Message]` internally
   - On init, converts `[ToolDefinition]` → `[OpenAIToolDef]` (wrap each: `OpenAIToolDef(type: "function", function: .init(name: td.name, description: td.description, parameters: td.inputSchema))`)
   - On init, prepends system message: `Message(role: "system", content: systemPrompt, toolCalls: nil, toolCallId: nil)`
   - `appendUserMessage`: appends `Message(role: "user", content: text, toolCalls: nil, toolCallId: nil)`
   - `appendAssistantResponse`: converts response back to OpenAI format — builds `Message(role: "assistant", content: textContent, toolCalls: [...])`. For tool calls, serialize each `input: JSONValue` back to a JSON string for the `arguments` field. CRITICAL: Do NOT include any `reasoning_content` field.
   - `appendToolResults`: for EACH tool result, append a SEPARATE `Message(role: "tool", content: content, toolCalls: nil, toolCallId: id)` — OpenAI format requires individual tool result messages, NOT an array in one message.
   - `callWithRetry`: Same retry logic as Anthropic. Builds `OpenAIToolRequest`, calls `https://api.deepseek.com/v1/chat/completions` with `Bearer {apiKey}` auth. Decodes `OpenAIToolResponse`. Translates:
     - `finish_reason: "stop"` → `.endTurn`
     - `finish_reason: "tool_calls"` → `.toolUse`
     - `finish_reason: "length"` → `.maxTokens`
     - Other → `.other(value)`
   - For tool call arguments: parse `arguments` string → `JSONValue` via `JSONDecoder().decode(JSONValue.self, from: Data(arguments.utf8))`
   - `pricingPerToken`: returns from static pricing map

5. **Static pricing map** (at module level or as static let):
