With an inline install flow that uses GuidedInstaller:

```swift
var status = DependencyChecker().checkAll()
if !status.allRequired {
    print("Missing dependencies detected. Setting up now...\n")
    let installer = GuidedInstaller()
    if status.homebrew == nil {
        installer.installHomebrew()
        status = DependencyChecker().checkAll()
    }
    if status.homebrew != nil && status.wine == nil {
        installer.installWine()
        status = DependencyChecker().checkAll()
    }
    guard status.allRequired else {
        print("Error: Dependencies could not be installed automatically.")
        print("Try this: Run `cellar` for step-by-step setup guidance.")
        throw ExitCode.failure
    }
    print("\nDependencies installed successfully. Continuing with game installation...\n")
}
guard let wineURL = status.wine else {
    print("Error: Wine is not installed.")
    print("Try this: Run `cellar` for step-by-step setup guidance.")
    throw ExitCode.failure
}
```

Key decisions:
- Call GuidedInstaller methods directly — they already handle retry prompts and streaming output
- Re-check status after each install step (Homebrew must succeed before Wine can install)
- Don't attempt winetricks install here — it's only needed for specific games, not for the initial add flow
- If auto-install fails, fall back to pointing user to `cellar` (StatusCommand) for manual guidance
- The `var status` (mutable) pattern allows re-checking after each install

Also verify that GuidedInstaller has the methods we're calling (installHomebrew, installWine). Read the file first. If GuidedInstaller has an `installWinetricks()` method, do NOT call it here — winetricks is only needed reactively.

With dynamic BottleScanner resolution:

```swift
} else if let recipe = try RecipeEngine.findBundledRecipe(for: game) {
    let discovered = BottleScanner.scanForExecutables(bottlePath: bottleURL)
    if let found = BottleScanner.findExecutable(named: recipe.executable, in: discovered) {
        executablePath = found.path
    } else if let first = discovered.first {
        executablePath = first.path
    } else {
        print("Error: No executables found in bottle for '\(game)'.")
        print("Try this: Run `cellar add /path/to/installer.exe` to reinstall the game.")
        throw ExitCode.failure
    }
```

This is a surgical 4-line replacement. Behavior for Cossacks is unchanged (BottleScanner will still find the same exe at its actual installed path) but now works for any game that has a bundled recipe.

The `else` block at the end (line 45-47 "Error: No executable path stored...") stays as-is but ensure it also has a "Try this:" line (may already be handled by Plan 01 — check and add if missing).
