The run() body follows the brief exactly:
1. Create `var state = LoopState(...)` using provider values
2. `provider.appendUserMessage(initialMessage)`
3. `while state.iterationCount < maxIterations` loop with:

   - Budget halt check via `checkBudgetHalt()`

4. After loop: return .maxIterations

- `consecutiveContinuations` variable — gone
- All budget tracking vars (hasAlertedAt50, hasWarnedAt80, etc.) — gone (middleware)
- All spin detection vars (recentActionTools, hasSentPivotNudge, actionTools set) — gone (middleware)
- `estimatedCost()` local function — replaced by `state.estimatedCost`
- `makeResult()` local function — replaced by `state.makeResult()`
- The `canStop/consecutiveContinuations` tug-of-war in `.endTurn` case — replaced by immediate return
- `"STOP": true` string matching — replaced by `result.isStop`

4. `private func handleAPIError(_:state:) -> AgentLoopResult` — format error, emit, return apiError result.

5. `private mutating func handleTruncation(response:state:)` — three cases: incomplete tool use below ceiling (escalate or budget-override), incomplete at ceiling (continuation), text-only (continuation with maxTokens reset).
