Create `Sources/cellar/Core/DependencyChecker.swift`:

```
struct DependencyStatus {
    let homebrew: URL?      // path to brew binary, or nil
    let wine: URL?          // path to wine binary, or nil
    let gptk: Bool          // GPTK detected
    var allRequired: Bool { homebrew != nil && wine != nil }
}

struct DependencyChecker {
    func detectHomebrew() -> URL?
    func detectWine(brewPrefix: URL) -> URL?
    func detectGPTK() -> Bool
    func checkAll() -> DependencyStatus
}
```

Implementation details:
- `detectHomebrew()`: Check `/opt/homebrew/bin/brew` (ARM) then `/usr/local/bin/brew` (Intel) using `FileManager.default.fileExists(atPath:)`. Return first found.
- `detectWine(brewPrefix:)`: From brew path, derive bin directory (`brew.deletingLastPathComponent()`). Check for `wine64` then `wine` in that directory.
- `detectGPTK()`: Best-effort check at `/usr/local/bin/gameportingtoolkit` and `/opt/homebrew/bin/gameportingtoolkit`. Return true if either exists.
- `checkAll()`: Call all three, compose DependencyStatus. For Wine detection, pass detected Homebrew URL if present.

Tests: Use a protocol `FileExistenceChecker` with a default implementation using `FileManager` and a test mock. This allows testing detection logic without real filesystem dependencies. Alternatively, test the path-construction logic and document that full integration requires real Homebrew/Wine installed.

Create `Tests/cellarTests/DependencyCheckerTests.swift` with tests for:
- Path construction logic (ARM path checked before Intel)
- Wine binary resolution from brew prefix
- DependencyStatus.allRequired computed property
