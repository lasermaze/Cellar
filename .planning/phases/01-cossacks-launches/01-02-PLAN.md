Create `Sources/cellar/Core/GuidedInstaller.swift` with a `GuidedInstaller` struct providing two methods:

**`installHomebrew()`**:
- Print: "Installing Homebrew..."
- Run the official Homebrew install script: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`
- IMPORTANT: Use `Foundation.Process` with `Pipe` on both stdout and stderr. Attach `readabilityHandler` to stream Homebrew's output to the terminal in real-time (per user decision: "Stream Homebrew's own output in real-time during installation").
- Let Homebrew handle Xcode CLT detection (per user decision: "Let Homebrew handle Xcode CLT detection and prompts").
- After process exits, check `terminationStatus`. If non-zero: print error message, offer retry ("Installation failed. Retry? [y/n]"), and on "n" print manual install instructions (`/bin/bash -c "$(curl -fsSL ...)"`) as fallback.
- On success: print "Homebrew installed successfully."
- After `waitUntilExit()`, drain remaining pipe data (readabilityHandler EOF bug workaround per RESEARCH.md Pitfall 4).

**`installWine()`**:
- Print: "Adding Gcenx Wine tap..."
- Run `brew tap gcenx/wine` with real-time output streaming (same Pipe pattern).
- Then print: "Installing Wine (this may take a few minutes)..."
- Run `brew install --no-quarantine gcenx/wine/wine-crossover` with real-time streaming. The `--no-quarantine` flag is CRITICAL (per RESEARCH.md Pitfall 1: prevents Gatekeeper "damaged" error).
- Same error handling: check exit status, offer retry, manual fallback on "n".
- On success: print "Wine installed successfully."

Both methods should resolve the `brew` binary path using `DependencyChecker().detectHomebrew()` for the Wine install step (Homebrew must exist first). For the Homebrew install, use `/bin/bash` directly since brew doesn't exist yet.

Helper: Create a private `runStreamingProcess(executablePath:arguments:)` method that handles the Pipe + readabilityHandler + drain pattern to avoid code duplication. This helper should:
1. Create Process with Pipe on stdout + stderr
2. Attach readabilityHandler that writes to FileHandle.standardOutput/standardError
3. Run process and waitUntilExit
4. Drain remaining data from both pipes after exit
5. Return the termination status

Use `readLine()` for the retry prompt (synchronous terminal input).

**StatusCommand.run() flow** (per CONTEXT.md decisions):

1. Run `DependencyChecker().checkAll()` to get `DependencyStatus`.

2. Print dependency status with clear indicators:
   ```
   Cellar - Wine game launcher for macOS

   Dependencies:
     Homebrew: [installed path] or [not found]
     Wine:     [installed path] or [not found]
     GPTK:    [detected] or [not detected] (optional)
   ```

3. If `!status.allRequired` (missing deps):
   - If Homebrew missing: print explanation, ask "Install Homebrew now? [y/n]", call `GuidedInstaller().installHomebrew()` on "y". After install, re-check Homebrew.
   - If Wine missing (after Homebrew is confirmed): print explanation, ask "Install Wine now? [y/n]", call `GuidedInstaller().installWine()` on "y". After install, re-check Wine.
   - After all installs, re-run full `checkAll()` and print updated status.

4. If `status.allRequired` (all deps present):
   - Print: "All dependencies found. Run `cellar add /path/to/setup.exe` to get started."
   (Per user decision: after setup is complete, show status + next step)

Output style: informative — a few lines per action explaining what's happening (per user decision: "not minimal and not verbose").
