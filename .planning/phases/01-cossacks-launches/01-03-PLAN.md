**Sources/cellar/Core/WineProcess.swift** — Reusable Wine subprocess launcher:

Create a `WineProcess` struct with:

```
struct WineProcess {
    let wineBinary: URL         // full path to wine binary
    let winePrefix: URL         // WINEPREFIX directory

    /// Run a Wine command with WINEPREFIX set, streaming output to terminal and optionally to a log file.
    func run(binary: String, arguments: [String] = [], environment: [String: String] = [:], logFile: URL? = nil) throws -> Int32

    /// Run wineboot with --init to initialize a new prefix
    func initPrefix() throws

    /// Run wine regedit with a .reg file
    func applyRegistryFile(at path: URL) throws

    /// Kill wineserver for this prefix
    func killWineserver() throws
}
```

Implementation of `run()`:
1. Create `Foundation.Process` with `executableURL = wineBinary`
2. Set `process.arguments = [binary] + arguments` (pass args as array, NEVER shell string — per RESEARCH.md anti-pattern on Windows path escaping)
3. Build environment: start with `ProcessInfo.processInfo.environment`, set `WINEPREFIX = winePrefix.path`, merge any additional `environment` dict
4. Create Pipe for stdout and stderr
5. Attach `readabilityHandler` on both pipes that:
   - Writes data to `FileHandle.standardOutput` / `FileHandle.standardError` (real-time terminal)
   - If `logFile` is provided, also writes to the log file handle
6. Call `process.run()` (NOT the deprecated `launch()`)
7. Call `process.waitUntilExit()`
8. Drain remaining data from both pipes after exit (RESEARCH.md Pitfall 4: readabilityHandler EOF bug)
9. Close log file handle if opened
10. Return `process.terminationStatus`

For `initPrefix()`:
- Resolve `wineboot` binary: same directory as wine binary, named `wineboot`
- Set additional env: `WINEDLLOVERRIDES=mscoree,mshtml=` (suppress Gecko/Mono dialogs per RESEARCH.md Pitfall 5)
- Print: "Initializing Wine bottle (first-time setup, may take ~30 seconds)..."
- Run wineboot with `--init` argument, streaming output

For `applyRegistryFile(at:)`:
- Run `regedit` with the .reg file path as argument

For `killWineserver()`:
- Resolve `wineserver` binary in same directory as wine
- Run with `-k` argument

**Sources/cellar/Core/RecipeEngine.swift**:

```
struct RecipeEngine {
    /// Load a recipe from a JSON file path
    static func loadRecipe(from path: URL) throws -> Recipe

    /// Find bundled recipe by game ID. Looks in the recipes/ directory relative to the executable.
    static func findBundledRecipe(for gameId: String) throws -> Recipe?

    /// Apply a recipe to a bottle: registry edits + env vars. Returns the merged environment dict for launch.
    func apply(recipe: Recipe, wineProcess: WineProcess) throws -> [String: String]
}
```

`loadRecipe(from:)`:
- Read file data with `Data(contentsOf: path)`
- Decode with `JSONDecoder()` — use explicit CodingKeys on Recipe (from Plan 01), so no special decoder strategy needed
- Return decoded Recipe

`findBundledRecipe(for:)`:
- Look for `{gameId}.json` in a `recipes/` directory. For development, resolve relative to current working directory. For release, resolve relative to the executable's bundle/directory.
- Strategy: Try `Bundle.main.url(forResource: gameId, withExtension: "json", subdirectory: "recipes")` first. Fallback to `FileManager.default.currentDirectoryPath + "/recipes/{gameId}.json"`. The fallback covers `swift run` during development.
- Return nil if not found (no recipe for this game — relevant for Phase 2 AI generation)

`apply(recipe:wineProcess:)` — This is where transparency happens:
1. Print: "Applying recipe: \(recipe.name)"
2. For each `RegistryEntry` in `recipe.registry`:
   - Print: "  Registry: \(entry.description)"
   - Print each line of `entry.regContent` indented (the diff-like transparency per user decision)
   - Write `entry.regContent` to a temp file (UUID-named .reg in NSTemporaryDirectory)
   - Call `wineProcess.applyRegistryFile(at: tempFile)`
   - Delete temp file
3. For each key-value in `recipe.environment`:
   - Print: "  Setting \(key)=\(value)"
4. Print: "Recipe applied."
5. Return the merged environment dictionary: start with `recipe.environment`, this will be merged into the Wine process environment at launch time

The environment dict is returned (not applied now) because it needs to be set on the game launch process, not on a regedit process.
