<task type="auto">
  <name>Task 1: GitHub Actions release workflow</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/release.yml` triggered on push of tags matching `v*`.

The workflow runs on `macos-latest` (ARM64 since 2024) with `contents: write` permission:

1. **Checkout** — `actions/checkout@v4`
2. **Build universal binary** — `swift build -c release --arch x86_64 --arch arm64`. The output path is `.build/apple/Products/Release/cellar` (the `apple` directory is used for multi-arch builds, not an architecture-specific directory).
3. **Archive** — `tar -czf cellar-${VERSION}-macos.tar.gz -C .build/apple/Products/Release cellar` where VERSION is derived from `${GITHUB_REF_NAME#v}` (strip the `v` prefix from the tag).
4. **Upload to GitHub Releases** — Use `softprops/action-gh-release@v2` with `files: ${{ env.ARCHIVE }}`.
5. **Update Homebrew formula** — Use `NSHipster/update-homebrew-formula-action@main` with:
   - `repository: ${{ github.repository }}`
   - `tap: <org>/homebrew-cellar` (use a placeholder `cellar-community/homebrew-cellar` — the user will update the org)
   - `formula: Formula/cellar.rb`
   - `GH_PERSONAL_ACCESS_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}`

Export `ARCHIVE` and `VERSION` as environment variables via `$GITHUB_ENV` for use across steps.

IMPORTANT: Do NOT use `macos-13` — `macos-latest` is ARM64 and `swift build --arch x86_64 --arch arm64` produces a valid fat binary from either architecture.
  </action>

<task type="auto">
  <name>Task 2: Homebrew tap formula with post_install .app creation</name>
  <files>Formula/cellar.rb</files>
  <action>
Create `Formula/cellar.rb` as a template that will live in the tap repo (`<org>/homebrew-cellar`). Store it in the main repo for reference and CI consumption.

The formula must include:

1. **Metadata** — `desc` ("AI-powered Wine launcher for old Windows games on macOS"), `homepage` (GitHub repo URL), `url` pointing to GitHub Releases download pattern, `sha256` placeholder (`"PLACEHOLDER"`), `version` placeholder.

2. **Bottle block** — Use `cellar :any_skip_relocation` since the binary has no dynamic library dependencies. Include `root_url` pointing to GitHub Releases. Include placeholder sha256 entries for `arm64_sequoia` and `sequoia`.

3. **install method** — Simply `bin.install "cellar"` (the archive contains just the binary).

4. **post_install** — Two things:
   a. Ad-hoc codesign: `system "codesign", "--sign", "-", bin/"cellar"`
   b. Create minimal `Cellar.app` in `libexec/`:
      - `libexec/"Cellar.app/Contents/MacOS"` directory
      - Write `Info.plist` with CFBundleExecutable=CellarLauncher, CFBundleIdentifier=dev.cellar.launcher, CFBundleName=Cellar, LSUIElement=true (hides from Dock)
      - Write `CellarLauncher` shell script that: checks if port 8080 is in use via `lsof -i TCP:8080 -s TCP:LISTEN -t`, starts `cellar serve` in background if not, polls up to 10 seconds (20 iterations x 0.5s) for the port to open, then runs `open http://127.0.0.1:8080`
      - Use `#{opt_bin}/cellar` for the binary path (NOT hardcoded `/opt/homebrew/bin/cellar`) — this resolves correctly on both Intel and Apple Silicon
      - chmod 0755 the launcher script

5. **caveats** — Tell user to run `cellar install-app` to copy Cellar.app to ~/Applications for double-click launching.

6. **test block** — `system "#{bin}/cellar", "--version"`
  </action>
  <verify>
    <automated>ruby -c Formula/cellar.rb && echo "PASS: valid Ruby syntax" && grep -q "codesign" Formula/cellar.rb && echo "PASS: ad-hoc signing" && grep -q "LSUIElement" Formula/cellar.rb && echo "PASS: .app bundle" && grep -q "opt_bin" Formula/cellar.rb && echo "PASS: correct bin path"</automated>
  </verify>
  <done>Formula/cellar.rb is valid Ruby, includes bottle block, post_install with codesign and .app creation, caveats referencing install-app, and test block</done>
</task>
