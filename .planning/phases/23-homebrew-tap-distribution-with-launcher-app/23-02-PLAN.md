  <action>
Create `Sources/cellar/Commands/InstallAppCommand.swift` following the existing command pattern (ParsableCommand, static configuration, mutating func run).

Configuration:
- `commandName: "install-app"`
- `abstract: "Install Cellar.app to ~/Applications for double-click launch"`

The `run()` method:

1. **Locate the .app bundle** — The binary is at some path like `/opt/homebrew/bin/cellar`. The libexec .app would be at `../libexec/Cellar.app` relative to the bin directory. Discover the current executable path using `CommandLine.arguments[0]` resolved via FileManager, then navigate up from its parent directory:
   - Resolve symlinks on the executable path (Homebrew symlinks from bin/ to Cellar/version/bin/)
   - From the resolved binary location, go up to the Homebrew prefix (e.g., `/opt/homebrew/Cellar/cellar/1.0.0/bin/cellar` -> navigate to `/opt/homebrew/Cellar/cellar/1.0.0/libexec/Cellar.app`)
   - Alternatively and more robustly: use the binary's resolved parent directory, go up one level, then into `libexec/Cellar.app`
   - If the .app is not found at that location, print an actionable error: "Cellar.app not found. This command works after installing via Homebrew (brew install cellar). If you built from source, the .app is created by the Homebrew formula's post_install step."

2. **Determine destination** — `~/Applications/Cellar.app`. Use `FileManager.default.homeDirectoryForCurrentUser` to get home directory. If `~/Applications` does not exist, create it with `FileManager.default.createDirectory`.

3. **Copy the .app** — If `~/Applications/Cellar.app` already exists, remove it first (to update). Then copy using `FileManager.default.copyItem`. Print success message: "Cellar.app installed to ~/Applications/Cellar.app\nDouble-click it to start the Cellar web UI."

4. **Error handling** — Catch FileManager errors and print actionable messages. Use `throw ExitCode.failure` or `print()` + `throw ExitCode.failure` pattern consistent with other commands.

Use `@Sendable` and Swift 6 concurrency annotations as needed to match the project's Swift 6 setting.
