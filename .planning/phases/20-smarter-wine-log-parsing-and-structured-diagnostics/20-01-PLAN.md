**New patterns by subsystem (use contains() for simple checks, NSRegularExpression for captures):**

Audio (category: .audio):
1. `err:dsound:` — "DirectSound error: {function extracted after last colon}"
   Fix: `.setDLLOverride("dsound", "n,b")`
2. `err:alsa:` or `err:mmdevapi:` — "Audio backend error ({alsa|mmdevapi})"
   Fix: nil (platform-specific)
3. `err:dsound` + contains "no driver" or "80004001" — "DirectSound: no audio driver"
   Fix: `.setDLLOverride("dsound", "n,b")`

Input (category: .input):
1. `err:dinput:` — "DirectInput error"
   Fix: `.installWinetricks("dinput8")`
2. `err:xinput` or `fixme:xinput` + contains "GetCapabilities" — "XInput device not found"
   Fix: `.setEnvVar("SDL_JOYSTICK_DISABLED", "1")`

Font (category: .font):
1. `err:font:` + contains "freetype" or "cannot find" + "FreeType" — "FreeType font loading error"
   Fix: nil (Wine installation issue)
2. `err:gdi:` or `err:font:` (catch-all after freetype check) — "GDI/font rendering error"
   Fix: `.installWinetricks("corefonts")`

Memory (category: .memory):
1. `err:seh:` + contains "Unhandled exception" + "page fault" — "Access violation (page fault)"
   Fix: `.installWinetricks("vcrun2019")` (most common cause for old games)
2. `err:virtual:virtual_setup_exception` or `err:virtual:virtual_handle_signal` — "Memory protection error"
   Fix: `.setEnvVar("WINE_CPU_TOPOLOGY", "1:0")`
3. `err:ntdll:RtlpWaitForCriticalSection` — "Deadlock detected (critical section timeout)"
   Fix: nil

**Positive success signals (add WineSuccess entries):**
- `trace:ddraw:` + contains "init" or "initialized" -> graphics success: "DirectDraw initialized"
- `trace:dsound:` + contains "open" or "success" -> audio success: "Audio device opened"
- `trace:dinput:` + contains "acquired" or "created" -> input success: "DirectInput device acquired"
- `d3d_adapter_init` or `wined3d` + contains "GL_RENDERER" -> graphics success: "Direct3D adapter initialized"

**Update existing patterns** to use the new category routing (existing 5 patterns stay, just route through WineDiagnostics.addError instead of appending to array).

**Causal chain detection** — post-pass after all errors collected:
Build a small hardcoded dependency map:
```swift
private static let dllToChanelMap: [String: String] = [
    "d3d9.dll": "d3d", "d3d9": "d3d",
    "d3d11.dll": "d3d", "d3d11": "d3d",
    "d3dx9": "d3d",
    "ddraw.dll": "ddraw", "ddraw": "ddraw",
    "dsound.dll": "dsound", "dsound": "dsound",
    "dinput.dll": "dinput", "dinput8.dll": "dinput",
]
```
For each missingDLL error, check if any other error's detail contains the mapped channel. If so, create a CausalChain: rootCause=the missing DLL error, downstreamEffects=matching errors, summary="missing {dll} caused {downstream detail}".

**Noise filtering** — line-by-line during parsing:
1. Track `subsystemsWithErrors: Set<String>` — add the Wine channel name (e.g. "dsound", "d3d", "font") whenever an error is detected.
2. For fixme: lines, extract the channel (second component after splitting on ":"). If channel NOT in subsystemsWithErrors, increment filteredFixmeCount. If channel IS in subsystemsWithErrors, keep the line (do not filter).
3. macOS harmless warn: allowlist — filter lines containing any of:
   - "Could not determine screen saver state"
   - "RtlSetHeapInformation"
   - "GetSystemFirmwareTable"
   - "DrvDocumentEvent"
   - "parse_depend_manifests"
   - "parse_assembly_identity"
   Increment filteredHarmlessWarnCount for each filtered.

**Filtered log generation:**
Add a second static method: `static func filteredLog(_ stderr: String, diagnostics: WineDiagnostics) -> String` that returns stderr with noise lines removed but all err:/warn:/trace: lines from subsystems-with-errors kept. This will be used by read_log in Plan 02.

Parse line-by-line (stderr.components(separatedBy: "\n")) for performance on large logs (50K+ lines).
