**CellarPaths.swift — add three new members:**

```swift
/// The collective memory repository identifier (owner/repo).
/// Reads CELLAR_MEMORY_REPO from process environment, falls back to default.
static var memoryRepo: String {
    ProcessInfo.processInfo.environment["CELLAR_MEMORY_REPO"] ?? defaultMemoryRepo
}

/// Directory for cached collective memory entries.
static let memoryCacheDir: URL = base.appendingPathComponent("cache/memory")

/// Cache file path for a given game slug.
static func memoryCacheFile(for slug: String) -> URL {
    memoryCacheDir.appendingPathComponent("\(slug).json")
}
```

Place these near the existing `defaultMemoryRepo` property. `base` is the existing `~/.cellar` base URL.

**CollectiveMemoryService.swift — rewrite fetchBestEntry:**

1. **Remove auth check entirely** — delete the `GitHubAuthService.shared.getToken()` call and the `guard case .token` block (lines 23-27).

2. **Replace `GitHubAuthService.shared.memoryRepo`** with `CellarPaths.memoryRepo` on line 40.

3. **Remove Authorization header** — delete the line `request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")` (line 46). Keep the Accept and X-GitHub-Api-Version headers.

4. **Add cache-first logic** before the network fetch (after building the slug, before the URLRequest):
   - Build cache file path: `let cacheFile = CellarPaths.memoryCacheFile(for: slug)`
   - Check if cache is fresh (file exists and modification date < 3600 seconds ago)
   - If fresh, read data from cache file and jump to the decode+rank+format logic
   - Add a private static helper `isCacheFresh(_ url: URL) -> Bool` that checks `FileManager.default.attributesOfItem(atPath:)[.modificationDate]`
   - Add a private static helper `loadFromCache(_ url: URL) -> Data?` that reads `try? Data(contentsOf: url)`

5. **Add cache-write after successful 200 fetch:**
   ```swift
   // Cache the response
   try? FileManager.default.createDirectory(at: CellarPaths.memoryCacheDir, withIntermediateDirectories: true)
   try? data.write(to: cacheFile, options: .atomic)
   ```

6. **Add stale-cache fallback on 403/429 or network error:**
   - In the network error handler (where performFetch returns nil), try `loadFromCache(cacheFile)` before returning nil
   - In the HTTP status handler, when statusCode is 403 or 429, try loading from cache (even if stale) before returning nil

7. **Refactor the decode+rank+format logic** into a private static helper that both the cache path and network path can call:
   ```swift
   private static func decodeAndFormat(
       data: Data,
       gameName: String,
       localWineVersion: String,
       localFlavor: String
   ) -> String?
   ```
   Move the existing decode, filter, score, rank, format logic (steps 6-11) into this helper. Both the fresh-cache path and the 200-response path call it.
