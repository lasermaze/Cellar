**Env interface:**
```typescript
export interface Env {
  GITHUB_APP_PEM: string;
  GITHUB_APP_ID: string;
  GITHUB_INSTALLATION_ID: string;
  CELLAR_MEMORY_REPO: string;
}
```

**Default export:** fetch handler that routes POST /api/contribute to handleContribute(), returns 404 for anything else. Add CORS headers (Access-Control-Allow-Origin: *, Allow-Methods: POST, OPTIONS) and handle OPTIONS preflight.

**handleContribute(request, env):**
1. Check Content-Length <= 51200 (50KB). Return 413 if exceeded.
2. Parse JSON body. Return 400 on parse failure.
3. Extract `entry` field from body. Return 400 if missing.
4. **Validate entry** (mirror Swift sanitizeEntry):
   - Filter config.environment keys against ALLOWED_ENV_KEYS constant (the 13 keys from interfaces block). Drop unknown keys silently.
   - Validate config.dllOverrides: each mode must be in VALID_DLL_MODES set. Drop entries with invalid modes.
   - Validate config.registry: each key must start with "HKEY_CURRENT_USER\\" or "HKEY_LOCAL_MACHINE\\". Drop entries with other prefixes.
   - Truncate all string fields to their max lengths (env values: 200, dll: 50, source: 100, registry key: 200, valueName: 100, data: 200).
   - Cap launchArgs at 5 entries, each 100 chars.
   - Filter setupDeps: accept only strings matching /^[a-z0-9_]{1,50}$/.
   - Validate top-level required fields exist: gameId (string), gameName (string), config (object), environment (object with arch, wineVersion, macosVersion, wineFlavor), environmentHash (string). Return 400 if any missing.
5. **Rate limit:** Use a simple in-memory Map keyed by CF-Connecting-IP header. Track timestamps of last 10 writes. If 10 writes in the last hour, return 429. (Note: in-memory state resets on Worker restart, which is acceptable for this scale.)
6. **Get GitHub installation token:**
   - Call makeJWT(env.GITHUB_APP_PEM, env.GITHUB_APP_ID) to get a JWT
   - POST to https://api.github.com/app/installations/{GITHUB_INSTALLATION_ID}/access_tokens with JWT as Bearer token
   - Extract token from response
7. **GET+merge+PUT to GitHub Contents API:**
   - GET repos/{repo}/contents/entries/{slug}.json (Accept: application/vnd.github+json)
   - If 200: decode sha + content (base64), decode entries array, find entry by environmentHash — if match, increment confirmations + update lastConfirmed; else append new entry
   - If 404: create new file with [entry]
   - Encode merged array as base64, PUT with message and sha (if updating)
   - If 409 conflict: retry once (re-GET, re-merge, re-PUT)
8. Return { "status": "ok" } on success, { "status": "error", "message": "..." } on failure.

**makeJWT(pemKey, appId) helper:**
- Strip PEM headers (both "RSA PRIVATE KEY" and "PRIVATE KEY" formats), remove whitespace
- Decode base64 to ArrayBuffer
- Import as PKCS8 key via crypto.subtle.importKey with RSASSA-PKCS1-v1_5 / SHA-256
- Build JWT header: { alg: "RS256", typ: "JWT" }
- Build JWT payload: { iss: appId, iat: now - 60, exp: now + 510 }
- Sign with crypto.subtle.sign, encode as base64url
- Return header.payload.signature

**slugify(name) helper:** lowercase, replace non-alphanumeric with hyphens, collapse consecutive hyphens, trim leading/trailing hyphens. Must match the Swift slugify() behavior.

All constants (ALLOWED_ENV_KEYS, VALID_DLL_MODES, ALLOWED_REGISTRY_PREFIXES) defined as module-level const arrays/sets at the top of the file.
