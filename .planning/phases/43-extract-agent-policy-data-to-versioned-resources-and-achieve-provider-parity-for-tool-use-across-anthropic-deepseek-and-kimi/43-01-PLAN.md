Create `Sources/cellar/Core/PolicyResources.swift`. Public surface:

```swift
import Foundation

enum PolicyError: Error, CustomStringConvertible {
    case missingResource(String)
    case schemaVersionMismatch(file: String, expected: Int, got: Int)
    case malformedFrontmatter(String)
    case decodingError(file: String, underlying: Error)
    var description: String { /* readable */ }
}

struct PolicyResources {
    let systemPrompt: String              // body of system_prompt.md (frontmatter stripped)
    let engineDefinitions: [EngineDefinition]   // existing struct from EngineRegistry.swift
    let dllRegistry: [KnownDLL]                  // existing struct from KnownDLLRegistry.swift
    let envAllowlist: Set<String>
    let registryAllowlist: [String]
    let toolSchemas: [String: JSONValue]   // key = AgentToolName.rawValue

    static let shared: PolicyResources = {
        do { return try PolicyResources() }
        catch { fatalError("PolicyResources failed to load: \(error)") }
    }()

    private init() throws { /* see implementation below */ }
}
```

Implementation:
1. **Markdown frontmatter parser** (file-private free function):
   ```swift
   func parsePolicyFrontmatter(_ raw: String) throws -> (version: Int, body: String)
   ```
   - Splits by `\n`. If first line is not `---`, throw `.malformedFrontmatter`.
   - Finds closing `---`. Scans middle for line starting with `schema_version:`. Parses int.
   - Returns `(version, joined body lines)`. Throws on missing/malformed.
2. **Versioned JSON wrapper**:
   ```swift
   private struct VersionedFile<Payload: Decodable>: Decodable {
       let schemaVersion: Int
       enum CodingKeys: String, CodingKey { case schemaVersion = "schema_version" }
       // Payload decoded via secondary container in custom init using userInfo to carry expectedKey,
       // OR define one wrapper struct per resource that flattens schema_version + the payload key.
   }
   ```
   Simpler: define one small Decodable per JSON file, e.g.:
   ```swift
   private struct EnginesFile: Decodable { let schema_version: Int; let engines: [EngineDefinitionFile] }
   private struct EngineDefinitionFile: Decodable { /* snake_case keys */ ... }
   ```
   Map `EngineDefinitionFile` → `EngineDefinition` after decode. Same for `KnownDLLFile` → `KnownDLL`. Keep mapping in this file only.
3. **`init() throws`** loads in order:
   - `system_prompt.md` via `Bundle.module.url(forResource: "policy/system_prompt", withExtension: "md")`. Parse frontmatter; if `version != 1` throw `.schemaVersionMismatch`.
   - For each `*.json`, helper `loadVersionedJSON<T: Decodable>(_ name: String, expectedVersion: Int) throws -> T` that uses `Bundle.module.url(forResource: "policy/<name>", withExtension: "json")`, decodes, version-checks, returns.
   - Build `engineDefinitions` and `dllRegistry` by mapping the file structs.
   - Build `toolSchemas` by re-decoding the `schemas` value as `[String: JSONValue]` (JSONValue is already Codable in the codebase).
4. **Failing modes**:
   - Missing file → `PolicyError.missingResource(...)` (not fatalError — `init` is `throws`; fatal is at `shared` site so dev sees the cause).
   - Wrong version → `PolicyError.schemaVersionMismatch(...)`.
   - Decode failure → `PolicyError.decodingError(...)`.

CRITICAL: do NOT add Codable to existing `EngineDefinition` or `KnownDLL` types in their original files (43-02 might do that as part of consolidation). For 43-01, keep mapping local to PolicyResources.swift via private "File" structs. This preserves a narrow blast radius.
