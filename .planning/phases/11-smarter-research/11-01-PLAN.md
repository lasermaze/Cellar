**Data models (all Codable):**

```swift
struct ExtractedEnvVar: Codable {
    let name: String      // "WINEDEBUG"
    let value: String     // "-all"
    let context: String   // "From WineHQ AppDB comment"
}

struct ExtractedRegistry: Codable {
    let path: String      // "HKCU\\Software\\Wine\\Direct3D"
    let value: String?    // raw match after path
    let context: String
}

struct ExtractedDLL: Codable {
    let name: String      // "ddraw"
    let mode: String      // "native"
    let context: String
}

struct ExtractedVerb: Codable {
    let verb: String      // "vcrun2019"
    let context: String
}

struct ExtractedINI: Codable {
    let file: String?     // "ddraw.ini"
    let key: String
    let value: String
    let context: String
}

struct ExtractedFixes: Codable {
    var envVars: [ExtractedEnvVar]
    var registry: [ExtractedRegistry]
    var dlls: [ExtractedDLL]
    var winetricks: [ExtractedVerb]
    var iniChanges: [ExtractedINI]

    var isEmpty: Bool {
        envVars.isEmpty && registry.isEmpty && dlls.isEmpty &&
        winetricks.isEmpty && iniChanges.isEmpty
    }

    static let empty = ExtractedFixes(envVars: [], registry: [], dlls: [], winetricks: [], iniChanges: [])

    mutating func merge(_ other: ExtractedFixes) {
        envVars += other.envVars
        registry += other.registry
        dlls += other.dlls
        winetricks += other.winetricks
        iniChanges += other.iniChanges
    }
}

struct ParsedPage {
    let textContent: String
    let extractedFixes: ExtractedFixes
}
```

**Protocol:**

```swift
protocol PageParser {
    func canHandle(url: URL) -> Bool
    func parse(document: Document, url: URL) throws -> ParsedPage
}
```

**Shared regex extraction function (module-level or static):**

`extractWineFixes(from text: String, context: String) -> ExtractedFixes`

Regex patterns to implement:
- **Env vars:** `(WINE(?:DEBUG|DLLOVERRIDES|_CPU_TOPOLOGY)|DXVK_\w+|MESA_\w+|STAGING_\w+)\s*=\s*([^\s,;]+)` — but skip WINEDLLOVERRIDES matches (handled separately)
- **WINEDLLOVERRIDES compound:** `WINEDLLOVERRIDES\s*=\s*["']?([^"'\n]+)` — parse the value string splitting on `;` then each part as `name=mode`
- **DLL overrides:** `(\w+(?:\.dll)?)\s*=\s*(native|builtin|n,b|b,n|n|b|disabled)` — only match when NOT inside a WINEDLLOVERRIDES= (to avoid double-counting). Ensure the dll name is reasonable (not generic words that happen to precede =native).
- **Winetricks verbs:** `winetricks\s+((?:\w+\s*)+)` — split matched group on whitespace, each word is a verb. Filter to reasonable verb names (lowercase alphanumeric, 2+ chars).
- **Registry paths:** `(HKCU|HKLM|HKEY_CURRENT_USER|HKEY_LOCAL_MACHINE)\\[\\A-Za-z0-9_ ]+`
- **INI changes:** Only extract `key=value` patterns when text contains a `.ini` or `.cfg` filename reference nearby (within ~200 chars). Capture the filename if detectable. Use pattern: detect `.ini`/`.cfg` filename, then extract `(\w+)\s*=\s*(\w+)` patterns near it.

**WineHQParser:**
- `canHandle`: URL host contains "appdb.winehq.org"
- `parse`:
  - Extract test results from `table.whq-table tr` rows (Wine version, rating)
  - Extract comments from `div.panel-forum .panel-body` elements
  - Run `extractWineFixes` on each comment body with context "WineHQ AppDB comment"
  - Build textContent from comments and test result summaries
  - Fallback: if no `.panel-forum` elements found, fall through to generic extraction

**PCGamingWikiParser:**
- `canHandle`: URL host contains "pcgamingwiki.com"
- `parse`:
  - Extract code blocks from `.mw-parser-output pre` and `.mw-parser-output code`
  - Extract table content from `.mw-parser-output table.wikitable`
  - Extract fix-related section content (headings containing "fix", "workaround", "improvement", "issue")
  - Run `extractWineFixes` on code blocks and table cells with context "PCGamingWiki"
  - Build textContent from article body `.mw-parser-output` paragraphs, headings, lists
  - Fallback: if `.mw-parser-output` not found, fall through to generic extraction

**GenericParser:**
- `canHandle`: Always returns true (fallback)
- `parse`:
  - Extract `pre`, `code`, and `table` elements
  - Run `extractWineFixes` on extracted element text with context "Web page"
  - Build textContent from document body text via `doc.body()?.text()` (SwiftSoup handles entity decoding)
  - Truncate textContent to 8000 characters

**Helper function for all parsers:**

`static func selectParser(for url: URL) -> PageParser` — returns first matching from `[WineHQParser(), PCGamingWikiParser(), GenericParser()]`.

**Important implementation notes:**
- All SwiftSoup usage is synchronous within parse() — Document/Element never cross isolation boundaries
- Use `try? element.text()` for resilient extraction — individual element failures should not crash the parser
- Deduplicate extracted fixes (same name+value env var appearing in multiple comments)
