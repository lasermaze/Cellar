```swift
private init(existingFileURL: URL) {

    self.encoder = JSONEncoder()
    self.encoder.outputFormatting = .sortedKeys
}
```

```swift
let eventLogResume: String?

}
```

**Modify the contextParts assembly** to prefer event log resume:

```swift
if let eventLogResume = eventLogResume {
    contextParts.append(eventLogResume)

}
```

This replaces the existing `if let previousSession = previousSession { contextParts.append(...) }` block. The `previousSession == nil` guard on DiagnosticRecord should change to check both are nil:

```swift
if eventLogResume == nil && previousSession == nil,
```

```swift
// Clean up event log on successful session
if let oldLog = AgentEventLog.findMostRecent(gameId: gameId) {
```

**Keep SessionHandoff.write() unchanged** on the failure path — it still writes handoff data on budget/iterations/error stops. The event log is already written by the EventLogger middleware during the session, so no additional write is needed.

**Keep SessionHandoff.delete() and SessionHandoff.read()** — they still clean up on read and delete on success. SessionHandoff is the fallback when no event log exists.
