## Human Verification Required

### 1. Reactive dep install — real installer failure path

**Test:** Run `cellar add` with a game installer that requires a winetricks dep (e.g. dotnet48) in a fresh bottle where the dep is absent.
**Expected:** Installer runs first; fails with exit code != 0; WineErrorParser diagnoses the missing dep; WinetricksRunner installs it; installer retries and succeeds.
**Why human:** Cannot programmatically simulate a Wine installer failure followed by dep installation — requires an actual Wine bottle and Windows installer binary.

### 2. Stale-output timeout trigger

**Test:** Simulate or observe a winetricks run that produces no output for 5 minutes (e.g. by hanging dotnet48 install in an interactive dialog that -q was supposed to suppress).
**Expected:** After 300 seconds of silence, WinetricksRunner kills the process, runs `wineserver -k`, prints the stall message, and returns `timedOut: true`.
**Why human:** Requires either a real hanging winetricks install or manual timing injection; automated grep cannot verify runtime behavior.

### 3. LaunchCommand dep-install-then-retry path

**Test:** Launch a game whose bottle is missing a DLL that WineErrorParser can diagnose (e.g. mfc42.dll mapped to vcrun6). Verify the retry loop installs the dep and retries the same config before trying variant 2.
**Expected:** Attempt 1 fails; dep installed; attempt 2 is labeled "Attempt 2: Base recipe configuration..." (same config); only advances to variant on non-dep failure.
**Why human:** Requires a real Wine environment with a missing DLL and a configured WineErrorParser rule that triggers installWinetricks.

### 4. --force-proactive-deps flag end-to-end

**Test:** Run `cellar add installer.exe --force-proactive-deps` with a recipe that has setup_deps.
**Expected:** Deps are installed upfront before the installer runs; post-failure reactive block is skipped.
**Why human:** Requires an actual installer and recipe with setup_deps to verify the pre-install loop fires before wineProcess.run().
