Create `Sources/cellar/Models/AIModels.swift` with:

1. `AIProvider` enum with cases `.anthropic(apiKey: String)`, `.openai(apiKey: String)`, `.unavailable`
2. `AIServiceError` enum conforming to `Error`: `httpError(statusCode: Int)`, `decodingError(String)`, `unavailable`, `allRetriesFailed`
3. `AIDiagnosis` struct: `explanation: String`, `suggestedFix: WineFix?`
4. `AIResult<T>` enum: `.success(T)`, `.unavailable`, `.failed(String)` (do NOT shadow Swift.Result -- use `AIResult` name)
5. Anthropic Codable types (per RESEARCH.md Pattern 3):
   - `AnthropicRequest`: model, max_tokens, system, messages array (use CodingKeys for snake_case)
   - `AnthropicResponse`: content array of ContentBlock (type + text), computed `firstText`
6. OpenAI Codable types:
   - `OpenAIRequest`: model, messages array, response_format (use CodingKeys for snake_case)
   - `OpenAIResponse`: choices array with nested message.content, computed `firstContent`

Create `Sources/cellar/Core/AIService.swift` with:

1. `static func detectProvider() -> AIProvider` — check `ProcessInfo.processInfo.environment` for `ANTHROPIC_API_KEY` first (prefer Claude per user decision), then `OPENAI_API_KEY`. Return `.unavailable` if neither set.

2. `private static func callAPI(request: URLRequest) throws -> Data` — uses `URLSession.shared.dataTask(with:completionHandler:)` with `DispatchSemaphore` bridge (NOT async variant). Check HTTP status >= 400 -> throw `httpError`. Use `URLSession.shared` (default background delegate queue, NOT custom session with main queue -- avoids semaphore deadlock per RESEARCH.md Pitfall 3).

3. `private static func withRetry<T>(maxAttempts: Int = 3, work: () throws -> T) throws -> T` — simple retry loop with `Thread.sleep(forTimeInterval: 1.0)` between attempts.

4. `static func diagnose(stderr: String, gameId: String) -> AIResult<AIDiagnosis>` —
   - Detect provider; return `.unavailable` if none
   - Truncate stderr to last 8000 characters (per RESEARCH.md Pitfall 4)
   - Build system prompt instructing AI to return JSON with keys: `explanation` (2-3 sentence plain English), `fix_type` (one of: installWinetricks, setEnvVar, setDLLOverride, none), `fix_arg1`, `fix_arg2`
   - Include the list of valid winetricks verbs from WineErrorParser in the system prompt: dotnet48, d3dx9, d3dx10, d3dx11_43, d3dcompiler_47, vcrun2019, xinput (prevents hallucinated verb names per RESEARCH.md Pitfall 2)
   - Route to Anthropic or OpenAI based on provider
   - For Anthropic: POST to `https://api.anthropic.com/v1/messages`, model `claude-haiku-4-5`, headers: `x-api-key`, `anthropic-version: 2023-06-01`, `content-type: application/json`
   - For OpenAI: POST to `https://api.openai.com/v1/chat/completions`, model `gpt-4o-mini`, headers: `Authorization: Bearer {key}`, `content-type: application/json`. Include `response_format: {type: "json_object"}`
   - Parse JSON response: extract `explanation`, map `fix_type` + args to `WineFix` enum. Validate verb against known-safe list; treat unrecognized as `none`.
   - Wrap in `withRetry`. On all failures, return `.failed(errorMessage)`.

5. `static func generateRecipe(gameName: String, gameId: String, installedFiles: [URL]) -> AIResult<Recipe>` —
   - Detect provider; return `.unavailable` if none
   - Build file context: take up to 50 file URLs, include filename only (not full path). Filter to .exe, .dll (from game dir only), .ini, .cfg extensions.
   - Build system prompt with the full Recipe JSON schema (all fields from Recipe.swift with snake_case keys). Instruct AI to generate a recipe JSON. Include an example based on the Cossacks recipe structure.
   - Route to provider (same as diagnose)
   - Parse response as JSON, decode to Recipe via JSONDecoder. Validate `executable` is non-empty. If decode fails, return `.failed`.
   - Wrap in `withRetry`. On all failures, return `.failed(errorMessage)`.

6. `static func showAITipIfNeeded()` — check for sentinel file `~/.cellar/.ai-tip-shown`. If absent, print "Tip: Set ANTHROPIC_API_KEY or OPENAI_API_KEY for AI-powered diagnosis and recipe generation." and create the sentinel file (use `.withoutOverwriting` for atomic creation per RESEARCH.md Open Question 3).

Key constraints:
- Use `URLSession.shared` only (NOT custom session) to avoid semaphore deadlock
- Use `dataTask(with:completionHandler:)` (NOT async `data(for:)`) since commands are synchronous ParsableCommand
- All types must be Sendable-safe for Swift 6 (structs with let properties are fine)
- Never print raw AI JSON to user — always extract structured fields
