<task type="auto">
  <name>Task 2: Add fputs stderr error logging to GitHub services</name>
  <files>
    Sources/cellar/Core/CollectiveMemoryService.swift,
    Sources/cellar/Core/CollectiveMemoryWriteService.swift,
    Sources/cellar/Core/GitHubAuthService.swift
  </files>
  <action>
Add `fputs(..., stderr)` error logging before every silent `return nil` or swallowed error in these three files. Use the format `[ServiceName] Descriptive message\n`.

**CollectiveMemoryService.swift — `fetchBestEntry()`:**
Instrument these failure paths (find each `return nil` and add fputs before it):
- Auth check failure: `fputs("[CollectiveMemoryService] Auth unavailable — skipping collective memory fetch\n", stderr)` — but only if this isn't already an expected path (check if auth unavailable is normal degradation; if so, use a lighter log or skip)
- Wine detection failure: `fputs("[CollectiveMemoryService] Could not detect Wine version for environment fingerprint\n", stderr)`
- Network error from performFetch/async fetch: `fputs("[CollectiveMemoryService] Network error fetching collective memory for '\(gameName)'\n", stderr)`
- Non-200 status codes (EXCEPT 404 — 404 means no entry exists, which is normal): `fputs("[CollectiveMemoryService] HTTP \(statusCode) fetching collective memory for '\(gameName)'\n", stderr)`
- JSON decode failure: `fputs("[CollectiveMemoryService] Failed to decode collective memory JSON for '\(gameName)'\n", stderr)`

**CollectiveMemoryWriteService.swift — `pushEntry()` and `performRequest()`/async fetch:**
- Network error: `fputs("[CollectiveMemoryWriteService] Network error during push for '\(gameId)'\n", stderr)`
- Non-2xx response on PUT: `fputs("[CollectiveMemoryWriteService] HTTP \(statusCode) on push for '\(gameId)'\n", stderr)`
- JSON encode failure: `fputs("[CollectiveMemoryWriteService] Failed to encode entry for '\(gameId)'\n", stderr)`
- 409 conflict (if not already retried): `fputs("[CollectiveMemoryWriteService] Conflict on push for '\(gameId)' — retrying\n", stderr)` (this is informational, not an error)

**GitHubAuthService.swift — `performHTTPRequest()` and `getToken()`:**
- Network error in HTTP request: `fputs("[GitHubAuthService] Network error: \(error.localizedDescription)\n", stderr)`
- Non-2xx response: `fputs("[GitHubAuthService] HTTP \(statusCode) from GitHub API\n", stderr)`
- Token decode failure: `fputs("[GitHubAuthService] Failed to decode installation token response\n", stderr)`
- JWT signing failure: `fputs("[GitHubAuthService] Failed to sign JWT: \(error.localizedDescription)\n", stderr)`

**Important:**
- Do NOT change any return types or control flow — only ADD fputs before existing returns
- Do NOT log on 404 in CollectiveMemoryService (normal "no entry" case)
- Do NOT log when auth is `.unavailable` — that's expected graceful degradation, not an error. Only log if auth was expected to work but failed.
- Use `fputs(message, stderr)` not `print(message)` — stderr keeps it out of normal user output
  </action>
