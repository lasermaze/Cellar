    **File 2: `Sources/cellar/Core/KnowledgeStore.swift`**

    Define the protocol exactly as locked in CONTEXT.md:
    ```swift
    protocol KnowledgeStore: Sendable {
        func fetchContext(for gameName: String, environment: EnvironmentFingerprint) async -> String?
        func write(_ entry: KnowledgeEntry) async
        func list(filter: KnowledgeListFilter) async -> [KnowledgeEntryMeta]
    }

    enum KnowledgeStoreContainer {
        nonisolated(unsafe) static var shared: KnowledgeStore = NoOpKnowledgeStore()
    }

    /// Default no-op so types compile in isolation. Plan 03 replaces this with a real
    /// KnowledgeStoreRemote at app startup. Kept private to this file for safety.
    private struct NoOpKnowledgeStore: KnowledgeStore {
        func fetchContext(for: String, environment: EnvironmentFingerprint) async -> String? { nil }
        func write(_: KnowledgeEntry) async {}
        func list(filter: KnowledgeListFilter) async -> [KnowledgeEntryMeta] { [] }
    }
    ```

    Why a container enum + nonisolated(unsafe) instead of `static var` on the protocol: protocols can't have static stored properties; mirroring `PolicyResources.shared` directly would require a class. The container is the lightest safe shim (Swift 6: nonisolated(unsafe) acceptable for a single-writer-at-startup pattern matching how `PolicyResources` is initialized). The setter is replaced exactly once in Plan 04 from AIService startup.
