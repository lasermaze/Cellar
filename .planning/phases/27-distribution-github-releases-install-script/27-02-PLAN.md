  <action>
Create `install.sh` at the repo root. The script must follow these exact specifications:

**Shebang and safety:**
```bash
#!/bin/bash
set -euo pipefail
```

**Color setup (conditional on terminal):**
- Define RED, GREEN, BOLD, RESET ANSI codes only when `[ -t 1 ]` (stdout is a terminal)
- When not a terminal (piped), set all color vars to empty strings

**Step 1 — OS detection:**
- Check `uname -s` equals "Darwin". If not, print error "Cellar requires macOS" to stderr and exit 1.

**Step 2 — Architecture detection:**
- Read `uname -m` for messaging (arm64 or x86_64). Universal binary works on both — this is informational only.

**Step 3 — Version resolution:**
- Set `INSTALL_DIR="${CELLAR_INSTALL_DIR:-$HOME/.cellar/bin}"`
- If `CELLAR_VERSION` is set and non-empty, use it as VERSION. Otherwise, fetch latest from GitHub API:
  ```
  curl -fsSL "https://api.github.com/repos/lasermaze/Cellar/releases/latest"
  ```
  Parse `.tag_name` with: `grep -o '"tag_name":"[^"]*"' | sed 's/"tag_name":"//;s/"//'`
- Normalize: `VERSION_NUM="${VERSION#v}"` (strip v-prefix for archive filename)
- Ensure VERSION has v-prefix for download URL: if VERSION doesn't start with v, prepend it

**Step 4 — Download to temp directory:**
- `TMPDIR=$(mktemp -d)` with `trap 'rm -rf "$TMPDIR"' EXIT`
- Archive name: `ARCHIVE="cellar-${VERSION_NUM}-macos.tar.gz"`
- Base URL: `https://github.com/lasermaze/Cellar/releases/download/${VERSION}`
- Download with `curl -fSL` (show progress, fail on error, follow redirects)

**Step 5 — Checksum verification:**
- Attempt to download `${ARCHIVE}.sha256` with `curl -fsSL` (silent)
- If download succeeds: verify with `(cd "$TMPDIR" && shasum -a 256 -c "${ARCHIVE}.sha256")` — MUST cd into TMPDIR first because shasum resolves relative filenames
- If download fails (no .sha256 file): print warning "checksum file not available, skipping verification" and continue

**Step 6 — Install binary:**
- `mkdir -p "$INSTALL_DIR"`
- `tar -xzf "$TMPDIR/$ARCHIVE" -C "$INSTALL_DIR"`
- `chmod +x "$INSTALL_DIR/cellar"`

**Step 7 — Remove quarantine:**
- `xattr -rd com.apple.quarantine "$INSTALL_DIR/cellar" 2>/dev/null || true`
- The `|| true` is CRITICAL — xattr returns exit code 1 when no quarantine attribute exists, which would abort under `set -e`

**Step 8 — PATH update (idempotent):**
- Detect shell RC file from `$SHELL`:
  - `*/zsh` -> `$HOME/.zshrc`
  - `*/bash` -> `$HOME/.bash_profile`
  - `*` -> `$HOME/.profile`
- Check if RC file already contains 'cellar/bin' using `grep -qF 'cellar/bin' "$RC" 2>/dev/null`
- If NOT present: append `export PATH="$HOME/.cellar/bin:$PATH"` with a leading newline using printf
- This ensures re-running the script doesn't duplicate PATH entries

**Step 9 — Smoke test:**
- Run `"$INSTALL_DIR/cellar" --help > /dev/null 2>&1`
- If fails: print troubleshooting message and exit 1

**Step 10 — Success message:**
- Print a success message showing:
  - Cellar version installed
  - Architecture
  - Install location
  - Getting started commands: restart shell or `source $RC`, then `cellar status`, `cellar add /path/to/installer.exe`, `cellar serve`

**Constraints:**
- Under 150 lines total
- No jq, python, node, or non-macOS-default tools
- No sudo
- Use `CELLAR_INSTALL_DIR` and `CELLAR_VERSION` env vars for customization
- `chmod +x install.sh` after creation
