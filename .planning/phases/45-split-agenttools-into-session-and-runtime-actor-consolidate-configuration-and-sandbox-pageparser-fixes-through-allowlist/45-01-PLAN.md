    In `Sources/cellar/Core/Tools/ResearchTools.swift`, insert the domain allowlist check inside `fetchPage` immediately after the URL parsing guard (the guard on line 146-148) and before the `var request = URLRequest(url: pageURL)` line:

    ```swift
    // ── Domain allowlist check (Phase 45) ──
    guard let host = pageURL.host else {
        return jsonResult(["error": "Domain not in allowlist", "url": urlStr,
                           "hint": "Use search_web to find relevant pages first"])
    }
    let domainAllowed = PolicyResources.shared.fetchPageAllowlist
        .contains(where: { host == $0 || host.hasSuffix(".\($0)") })

    }
    // ── end allowlist check ──
    ```

    The matching logic: `host == $0` handles exact domain match (e.g. "winehq.org" itself); `host.hasSuffix(".\($0)")` handles subdomains (e.g. "appdb.winehq.org" has suffix ".winehq.org"). This requires NO regex.

    In `Tests/cellarTests/PolicyResourcesTests.swift`, add a new test suite or test function verifying the subdomain suffix-matching logic (pure logic test, no network):

    ```swift
    @Test("fetchPageAllowlist subdomain matching logic")
    func fetchPageSubdomainMatching() {
        let allowlist: Set<String> = ["winehq.org", "githubusercontent.com", "github.com"]
        func isAllowed(_ urlStr: String) -> Bool {
            guard let host = URL(string: urlStr)?.host else { return false }
            return allowlist.contains(where: { host == $0 || host.hasSuffix(".\($0)") })
        }
        #expect(isAllowed("https://winehq.org/page") == true)
        #expect(isAllowed("https://appdb.winehq.org/page") == true)
        #expect(isAllowed("https://raw.githubusercontent.com/file") == true)
        #expect(isAllowed("https://example.com/page") == false)

    }
    ```
    The last assertion verifies that `hasSuffix(".\(entry)")` (with the dot prefix) prevents `evil-winehq.org` from matching `winehq.org`. This is the critical edge case.
