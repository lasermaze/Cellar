     /// Events emitted during agent loop execution for external consumers (web UI, logging).
     enum AgentEvent: Sendable {
         case iteration(number: Int, total: Int)
         case text(String)
         case toolCall(name: String)
         case toolResult(name: String, truncated: String)
         case cost(inputTokens: Int, outputTokens: Int, usd: Double)
         case budgetWarning(percentage: Int)
         case status(String)
         case error(String)
         case completed(AgentLoopResult)
     }
     ```

     ```

  3. Update the init to accept the callback with a default nil:

     init(

  4. Create a private helper that emits events AND prints (for CLI compatibility):
     ```swift
     private func emit(_ event: AgentEvent) {
         onOutput?(event)
         // Preserve CLI output when no callback is set (or always, for terminal users)
         switch event {
         case .iteration(let n, let total):
             print("[Agent iteration \(n)/\(total)]")
         case .text(let text):
             print("Agent: \(text)")
         case .toolCall(let name):
             print("-> \(name)")
         case .toolResult:
             break // not printed in CLI
         case .cost(_, _, let usd):
             // Only print cost at completion or budget warnings
             break
         case .budgetWarning(let pct):
             print("[Budget: \(pct)% used]")
         case .status(let msg):
             print(msg)
         case .error(let msg):
             print(msg)
         case .completed:
             break
         }
     }
     ```
     IMPORTANT: Check whether the existing print() statements should be kept as-is alongside the callback, or whether the callback should REPLACE them. The safest approach: when `onOutput` is nil, preserve all existing print() behavior exactly. When `onOutput` is non-nil, ALSO print (the web server ignores stdout, but `cellar launch` in terminal still needs output). So always print, and additionally call the callback.

     Actually, the simplest approach: replace each `print(...)` with `emit(...)` where the emit function always prints AND calls the callback. This ensures CLI behavior is 100% preserved.

  5. Replace every `print()` call in the `run()` method with the appropriate `emit()` call:
     - `print("[Agent iteration ...")` → `emit(.iteration(number: iterationCount, total: maxIterations))`

     - Status messages (continuation, max_tokens, truncation) → `emit(.status(...))`

  6. Make AgentLoopResult conform to Sendable if it doesn't already (required for AgentEvent.completed case). All its fields are value types so this should be automatic with Swift 6, but add explicit conformance if the compiler complains.
