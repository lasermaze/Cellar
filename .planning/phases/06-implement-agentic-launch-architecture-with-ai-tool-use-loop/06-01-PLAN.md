  <action>
Add the following types to AIModels.swift AFTER the existing types (preserve all existing types unchanged):

1. **`JSONValue`** — recursive indirect enum implementing Codable. Cases: `.string(String)`, `.number(Double)`, `.bool(Bool)`, `.null`, `.array([JSONValue])`, `.object([String: JSONValue])`. Use `singleValueContainer()` for both encode and decode. Add `Equatable` conformance. Add convenience accessors as extension: `asString`, `asBool`, `asNumber`, `asObject`, `asArray`, and `subscript(key: String) -> JSONValue?` for object field access.

CRITICAL for decode order: try Bool BEFORE Double (otherwise `true`/`false` decode as 1.0/0.0). Try String first, then Bool, then Double, then decodeNil, then Array, then Object.

2. **`MessageContent`** — enum with `.text(String)` and `.blocks([ContentBlock])` cases. Codable: decode tries String first (-> .text), falls back to [ContentBlock] array (-> .blocks). Encode: .text encodes as bare string, .blocks encodes as array.

3. **`ContentBlock`** (new, as a top-level enum — do NOT touch existing `AnthropicResponse.ContentBlock`) — tagged union with three cases:
   - `.text(String)`
   - `.toolUse(id: String, name: String, input: JSONValue)`
   - `.toolResult(toolUseId: String, content: String, isError: Bool)`

   Custom Codable: decode by reading `type` field ("text", "tool_use", "tool_result"). Encode with correct Anthropic API field names: `tool_use_id` for toolResult, `is_error` for isError (only encode if true), `input_schema` not needed here (that's on ToolDefinition).

4. **`ToolDefinition`** — struct with `name: String`, `description: String`, `inputSchema: JSONValue`. Encodable only (not decoded). CodingKey maps `inputSchema` to `input_schema`.

5. **`AnthropicToolRequest`** — struct with `model: String`, `maxTokens: Int`, `system: String?`, `messages: [Message]`, `tools: [ToolDefinition]?`. Inner `Message` struct with `role: String`, `content: MessageContent`. Encodable. CodingKey maps `maxTokens` to `max_tokens`.

6. **`AnthropicToolResponse`** — struct with `content: [ContentBlock]`, `stopReason: String`. Decodable. CodingKey maps `stopReason` to `stop_reason`. Note: this uses the NEW `ContentBlock` enum (not the existing `AnthropicResponse.ContentBlock`).

Name the new ContentBlock as `ToolContentBlock` to avoid collision with existing `AnthropicResponse.ContentBlock`. Update all references in the new types (MessageContent, AnthropicToolRequest.Message, AnthropicToolResponse) to use `ToolContentBlock`.

<task type="auto">
  <name>Task 2: Implement AgentLoop state machine</name>
  <files>Sources/cellar/Core/AgentLoop.swift</files>
  <action>
Create `Sources/cellar/Core/AgentLoop.swift` with a struct (or class — use class if mutable state + closure capture requires it for Swift 6 Sendable compliance).

**AgentLoop properties:**
- `apiKey: String` — Anthropic API key
- `model: String` — model ID (default "claude-sonnet-4-20250514")
- `tools: [ToolDefinition]` — tool definitions to send to API
- `systemPrompt: String` — system prompt for the agent
- `maxIterations: Int = 20` — max tool call iterations
- `maxTokens: Int = 4096` — max_tokens per API call
- Private mutable state: `messages: [AnthropicToolRequest.Message]`, `iterationCount: Int = 0`

**`run(initialMessage: String, toolExecutor: (String, JSONValue) -> String) -> AgentLoopResult`:**

AgentLoopResult is a simple struct: `let finalText: String`, `let iterationsUsed: Int`, `let completed: Bool` (true if end_turn, false if max iterations).

Loop logic:
1. Append initial message as user role with `.text(initialMessage)` content
2. While iterationCount < maxIterations:
   a. iterationCount += 1
   b. Call Anthropic API (build AnthropicToolRequest, encode, POST to https://api.anthropic.com/v1/messages with headers: x-api-key, anthropic-version: 2023-06-01, content-type: application/json)
   c. Decode AnthropicToolResponse
   d. Print any `.text` blocks to stdout (user sees agent thinking) — prefix with "Agent: " for clarity
   e. If stopReason == "end_turn": return finalText from text blocks, completed=true
   f. If stopReason == "tool_use":
      - Append assistant turn: `.blocks(response.content)` with role "assistant"
      - For each `.toolUse` block: print tool name ("-> toolName..."), call toolExecutor(name, input), collect `.toolResult` blocks
      - Append user turn: `.blocks(resultBlocks)` — tool_result blocks MUST be first (and only) in the content array per API requirement
   g. If stopReason is anything else (e.g., "max_tokens"): return with completed=false
3. Return max iterations result with completed=false

**HTTP method:** Reuse the same DispatchSemaphore + URLSession.shared + ResultBox pattern from AIService.callAPI. Copy it as a private method `callAPI(request: URLRequest) throws -> Data` in AgentLoop (AIService's version is private and cannot be reused directly). Alternatively, add an `internal` static method to AIService that AgentLoop can call — choose whichever keeps the code cleaner.

**Error handling:** If API call throws, catch and return AgentLoopResult with the error message as finalText, completed=false. Do not crash the agent loop on transient API errors.

**Terminal output during loop:** Print each tool call name before execution: `print("-> \(toolName)")`. Print text blocks from assistant as they arrive. This gives the user visibility into the agent's progress.
