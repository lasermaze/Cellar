  <name>Task 1: Implement AgentTools class with tool definitions and diagnostic tools</name>
  <files>Sources/cellar/Core/AgentTools.swift</files>
  <action>
Create `Sources/cellar/Core/AgentTools.swift` as a class (reference type for mutable state with closure capture in Swift 6).

**Class properties (injected via init):**
- `gameId: String`
- `entry: GameEntry`
- `executablePath: String` — resolved exe path (computed by LaunchCommand before constructing AgentTools)
- `bottleURL: URL`
- `wineURL: URL`
- `wineProcess: WineProcess`

**Mutable state:**
- `accumulatedEnv: [String: String] = [:]` — set_environment accumulates here
- `launchCount: Int = 0`
- `maxLaunches: Int = 8`
- `installedDeps: Set<String> = []` — track installed winetricks verbs
- `lastLogFile: URL? = nil` — most recent launch log for read_log

**Static property: `static let toolDefinitions: [ToolDefinition]`**

Define all 10 tools with JSON Schema input_schema objects. Each tool has a name, description, and inputSchema using JSONValue literals. Tool schemas:

1. `inspect_game`: no required params (game context is implicit)
2. `read_log`: optional `lines: number` (default 200)
3. `read_registry`: required `key_path: string` (e.g., "HKCU\\Software\\Wine\\DllOverrides"), optional `value_name: string`
4. `ask_user`: required `question: string`, optional `options: array of strings`
5. `set_environment`: required `key: string`, `value: string`
6. `set_registry`: required `key_path: string`, `value_name: string`, `data: string`
7. `install_winetricks`: required `verb: string`
8. `place_dll`: required `dll_name: string`, optional `target: string` (enum "game_dir"|"system32", default "game_dir")
9. `launch_game`: optional `extra_winedebug: string`
10. `save_recipe`: required `name: string`, optional `notes: string`

**`func execute(toolName: String, input: JSONValue) -> String` dispatch method:**
Switch on toolName, dispatch to private methods. Return JSON strings from all tools using a private helper `jsonResult(_ dict: [String: Any]) -> String` that uses JSONSerialization.

**Diagnostic tool implementations (this task):**

1. **inspectGame**: Use `/usr/bin/file` command (via Process) on executablePath to get PE32/PE32+ type. List files in game directory (gameDir = URL(fileURLWithPath: executablePath).deletingLastPathComponent()). Check bottle exists. List DLLs in system32 dir. Load recipe via RecipeEngine.findBundledRecipe. Return JSON with: exe_type, game_files (array), bottle_exists, system32_dlls (array), recipe (name/environment/registry or null).

2. **readLog**: Read last log file. If lastLogFile is nil, scan CellarPaths.logDir(for: gameId) for most recent .log file. Read file, return last 8000 chars of content. Return JSON with: log_content, log_file path.

3. **readRegistry**: Read registry file directly from bottle. Key paths starting with "HKCU" read from bottleURL/user.reg, "HKLM" from bottleURL/system.reg. Convert Windows key path separators (\\) to forward slashes as used in .reg files. Search for matching section header `[key_path]` and return values beneath it. Try UTF-8 encoding first, fall back to .windowsCP1252. Return JSON with: values (dict) or error.

4. **askUser**: Print question to stdout. If options array provided, print numbered options. Read from stdin via readLine(). Return JSON with: answer.

<task type="auto">
  <name>Task 2: Implement action and execution tools</name>
  <files>Sources/cellar/Core/AgentTools.swift</files>
  <action>
Add the remaining 6 tool implementations to AgentTools:

**Action tools:**

5. **setEnvironment**: Extract `key` and `value` from input. Store in `accumulatedEnv[key] = value`. Return JSON with: status "ok", key, value, current_env (full accumulatedEnv dict).

6. **setRegistry**: Extract `key_path`, `value_name`, `data` from input. Write a .reg file in the same format as WineActionExecutor.setRegistry (Windows Registry Editor Version 5.00 header, [keyPath], "valueName"=data). Write to temp file, call wineProcess.applyRegistryFile(at:), delete temp. Return JSON with: status "ok" or error.

7. **installWinetricks**: Extract `verb` from input. Validate against `AIService.validWinetricksVerbs` — if not in allowlist, return error "Verb 'X' not in allowed list. Allowed: [list]". Check `installedDeps` to skip duplicates. Find winetricks binary via DependencyChecker().checkAll().winetricks. Create WinetricksRunner and call install(verb:). Add to installedDeps on success. Return JSON with: status, verb, output (stderr tail).

8. **placeDLL**: Extract `dll_name` and optional `target` from input. Look up in KnownDLLRegistry.find(name:) — if not found, return error with message telling the agent the DLL is not in the known registry and the user should place it manually. Call DLLDownloader.downloadAndCache then .place into target dir (game dir or system32 based on target param). Apply requiredOverrides from KnownDLL by adding them to accumulatedEnv (WINEDLLOVERRIDES). Return JSON with: status, dll_path, applied_overrides.

**Execution tools:**

9. **launchGame**: Check launchCount < maxLaunches (return error if exceeded). Increment launchCount. Extract optional `extra_winedebug`. Build launch env: start with accumulatedEnv, merge extra_winedebug into WINEDEBUG. Create log file via CellarPaths.logFile. Create log directory. Call wineProcess.run(binary: executablePath, arguments: [], environment: env, logFile: logFile). Store logFile in lastLogFile. Parse errors via WineErrorParser.parse(result.stderr). Return JSON with: exit_code, elapsed_seconds, timed_out, stderr_tail (last 4000 chars), detected_errors (array of {category, detail, suggested_fix}), log_file path, launch_number.

10. **saveRecipe**: Extract `name` and optional `notes`. Build a Recipe from current state: id=gameId, name=name, version="1.0.0", source="ai-agent", executable=executablePath (just the filename, not full path), environment=accumulatedEnv, registry=[] (agent should have used set_registry for any needed), launchArgs=[], notes=notes. Call RecipeEngine.saveUserRecipe(). Return JSON with: status, recipe_path.

**Error handling pattern:** All tools catch errors with do/catch and return `jsonResult(["error": error.localizedDescription])`. Never throw from tool implementations — the agent loop expects String returns.
