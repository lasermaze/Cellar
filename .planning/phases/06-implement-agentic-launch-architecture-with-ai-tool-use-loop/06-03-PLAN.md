**New run() method (~50-70 lines):**

```swift
mutating func run() throws {
    // 1. Check dependencies (same as before)
    let status = DependencyChecker().checkAll()
    guard status.allRequired, let wineURL = status.wine else {
        print("Error: Wine is not installed.")
        print("Run `cellar` first to install dependencies.")
        throw ExitCode.failure
    }

    // 2. Find game (same as before)
    guard var entry = try CellarStore.findGame(id: game) else {
        print("Game not found. Run `cellar add /path/to/installer` first.")
        throw ExitCode.failure
    }

    // 3. Check bottle exists (same as before)
    let bottleManager = BottleManager(wineBinary: wineURL)
    guard bottleManager.bottleExists(gameId: game) else {
        print("Error: Bottle for '\(game)' not found.")
        throw ExitCode.failure
    }

    let bottleURL = CellarPaths.bottleDir(for: game)
    let wineProcess = WineProcess(wineBinary: wineURL, winePrefix: bottleURL)

    // 4. Resolve executable path (same logic as old step 6)
    let executablePath: String
    if let stored = entry.executablePath {
        executablePath = stored
    } else if let recipe = try RecipeEngine.findBundledRecipe(for: game) {
        let gogDir = bottleURL.path + "/drive_c/GOG Games/Cossacks - European Wars"
        executablePath = gogDir + "/" + recipe.executable
    } else {
        print("Error: No executable path stored and no recipe available.")
        throw ExitCode.failure
    }

    // 5. Try agent loop (requires Anthropic API key)
    switch AIService.runAgentLoop(
        gameId: game,
        entry: entry,
        executablePath: executablePath,
        wineURL: wineURL,
        bottleURL: bottleURL,
        wineProcess: wineProcess
    ) {
    case .success(let summary):
        // Agent completed — it handled launch, user interaction, and recipe saving
        print(summary)
        entry.lastLaunched = Date()
        try CellarStore.updateGame(entry)
        return

    case .unavailable:
        // No API key — fall back to recipe-only launch
        print("No AI API key configured. Launching with recipe defaults only.")
        try recipeFallbackLaunch(
            entry: &entry,
            executablePath: executablePath,
            wineProcess: wineProcess,
            wineURL: wineURL,
            bottleURL: bottleURL
        )

    case .failed(let msg):
        print("Agent error: \(msg)")
        print("Falling back to recipe-only launch...")
        try recipeFallbackLaunch(
            entry: &entry,
            executablePath: executablePath,
            wineProcess: wineProcess,
            wineURL: wineURL,
            bottleURL: bottleURL
        )
    }
}
```

**Add `recipeFallbackLaunch` private method (~40-50 lines):**

This is a simple single-launch path (no retry loop, no AI):
1. Load recipe via RecipeEngine.findBundledRecipe(for: game)
2. If recipe exists, apply it via RecipeEngine().apply(recipe:wineProcess:) to get recipeEnv
3. Prepare log file
4. Set up SIGINT handler (same wineserver -k pattern as before)
5. Launch via wineProcess.run(binary: executablePath, arguments: recipe?.launchArgs ?? [], environment: recipeEnv, logFile: logFile)
6. Cancel SIGINT handler
7. Run ValidationPrompt.run(gameId:elapsed:wineProcess:) for user feedback
8. Record result in entry.lastResult
9. CellarStore.updateGame(entry)

This fallback preserves the existing user experience for people without API keys. It is intentionally simple — one launch, one validation prompt, done. The old retry/escalation logic is gone (the agent replaces it).

**IMPORTANT:** Do NOT delete ValidationPrompt.swift. It is still used by the fallback path. The agent replaces it with ask_user for the AI-driven path.

**IMPORTANT:** Remove ALL of the old steps 4-10 from the original run() body (recipe loading, env config building, retry loop, escalation, repair report, winning config save). All of that is now handled by the agent.
