   ```swift
   let staleTimeout: TimeInterval = 300  // 5 minutes — per CONTEXT.md decision
   var didTimeout = false
   while process.isRunning {
       Thread.sleep(forTimeInterval: 2.0)
       if Date().timeIntervalSince(outputMonitor.lastOutputTime) > staleTimeout {
           print("\nGame launch has produced no output for \(Int(staleTimeout / 60)) minutes — assuming hung.")
           process.terminate()
           try? killWineserver()
           Thread.sleep(forTimeInterval: 1.0)
           didTimeout = true
           break
       }
   }
   ```

```swift
// MARK: - Generate Variants

/// Generate alternative recipe variants using AI after bundled variants are exhausted.
/// Returns up to 3 RetryVariant values with environment-only changes (no registry edits).
/// Returns .unavailable if no API key is configured.
static func generateVariants(
    gameId: String,
    gameName: String,
    currentEnvironment: [String: String],
    attemptHistory: [(description: String, envDiff: [String: String], errorSummary: String)]
) -> AIResult<AIVariantResult> {
    let provider = detectProvider()
    if case .unavailable = provider {
        return .unavailable
    }
    return _generateVariants(
        gameId: gameId, gameName: gameName,
        currentEnvironment: currentEnvironment,
        attemptHistory: attemptHistory, provider: provider
    )
}
```

**Private implementation** `_generateVariants()`:

1. Build the system prompt. Key constraints:
   - AI is a Wine compatibility expert generating alternative launch configurations
   - Output MUST be environment variables and WINEDLLOVERRIDES ONLY — no registry edits, no winetricks installs (per CONTEXT.md locked decision)
   - Return a JSON object with "reasoning" (string, 1-2 sentences explaining analysis) and "variants" (array of up to 3 objects, each with "description" (string) and "environment" (object of string key-value pairs))
   - Each variant should try a meaningfully different approach (not minor tweaks of the same idea)
   - Common Wine env vars to consider: WINEDLLOVERRIDES, MESA_GL_VERSION_OVERRIDE, MESA_GLSL_VERSION_OVERRIDE, WINED3D_DISABLE_CSMT, STAGING_SHARED_MEMORY, WINE_LARGE_ADDRESS_AWARE, WINEDEBUG, __GL_THREADED_OPTIMIZATIONS, DXVK_HUD, WINEFSYNC, WINEESYNC

2. Build user message with cumulative context:
   - Game ID and name
   - Current base environment (key=value pairs)
   - Attempt history: for each prior attempt, include description, env diff keys/values, and error summary CAPPED at 500 characters per entry (to prevent prompt token explosion — per RESEARCH.md anti-pattern)
   - "Generate up to 3 alternative configurations to try."

3. Call using `withRetry { try makeAPICall(provider:systemPrompt:userMessage:) }` — same pattern as diagnose/generateRecipe. Use maxTokens of 1500 (variants are compact JSON, higher than diagnose's 1024 to allow 3 variants).

**Private parser** `parseVariantsResponse(_ text: String) -> AIResult<AIVariantResult>`:

1. Use `extractJSON(from:)` to strip markdown fences
2. Parse as `[String: Any]` using JSONSerialization
3. Extract "reasoning" as String (default to "" if missing)
4. Extract "variants" as array of dictionaries

6. Map to `RetryVariant` instances
7. Cap at 3 variants (`.prefix(3)`)
8. Return `.success(AIVariantResult(variants: variants, reasoning: reasoning))`
9. Return `.failed(...)` if JSON structure is unparseable or variants array is empty
