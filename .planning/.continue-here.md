<context>
Three gaps identified from real-world testing with Deus Ex GOTY and Rayman 2:

1. LOOP RESILIENCE: Agent returns end_turn after first game failure instead of continuing to diagnose. System prompt says "loop back to Phase 2" but model doesn't follow. Also max_tokens truncation caused silent exits (now fixed).

2. DIALOG AWARENESS: Games show renderer selection dialogs (DirectX vs OpenGL) that block launch. Wine captures dialog text via WINEDEBUG=+msgbox (trace:msgbox lines in stderr) but WineErrorParser doesn't parse them. Agent is blind to these errors. macOS CGWindowListCopyWindowInfo can detect Wine windows without X11. Games like Deus Ex store renderer choice in INI files that can be pre-configured.

3. DEEP RESEARCH: DuckDuckGo search works but is shallow. PCGamingWiki has a Cargo API with structured game data and GOG product ID lookups. No cross-game engine pattern learning exists. System prompt lacks guidance on research depth.

Key architectural patterns from Lutris/Bottles/POL:
- Pre-configure to avoid dialogs entirely (best approach, all tools do this)
- Bottles uses YAML action manifests (override_dll, set_register_key, etc.)
- Lutris tags game processes with UUID env vars for reliable tracking
- CGWindowListCopyWindowInfo for macOS window detection (no X11 needed)

Proposed requirement categories: LOOP (resilience), DIAG (dialog/error detection), RESEARCH (deep research), ENGINE (game engine awareness)
</context>
